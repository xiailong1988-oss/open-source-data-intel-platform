# 任务：首页桌面端左右骨架修正与 Crucix 结构稳定

## 任务背景
- 用户在 2026-04-01 最新人工验证中明确指出：首页在桌面端出现“地图在上、右侧情报流在下”的上下堆叠。
- 当前问题不是缓存导致，而是首页组件 `HaidianCockpitPanel.vue` 中存在错误的桌面端响应式规则：
  - `@media (max-width: 1560px) { .haidian-cockpit__stage { grid-template-columns: 1fr; } }`
- 这条规则会把 1366 / 1440 / 1536 等正常桌面宽度误判为需要堆叠的小屏布局，直接破坏 Crucix 式“中央地图主舞台 + 右侧 signal rail”的核心关系。

## 本轮目标
1. 删除或重构桌面端错误的单列堆叠规则。
2. 强制首页在桌面端保持左右结构：
   - 左侧地图主舞台
   - 右侧 signal rail
3. 重新定义真正的小屏断点，仅在窄屏或移动端时才允许上下堆叠。
4. 保留当前已做对的交互逻辑：
   - 自动滚动只推动右侧流
   - 点击条目时才联动地图并打开详情

## 执行范围
- `src/components/dashboard/HaidianCockpitPanel.vue`
- `scripts/ui-smoke.mjs`
- `doc/rules/当前执行状态.md`

## 执行结果
- 已删除桌面端错误规则：`@media (max-width: 1560px) { .haidian-cockpit__stage { grid-template-columns: 1fr; } }`
- 已重新定义断点：
  - `1880px` 以下：仍保持左右结构，只微调右侧 rail 宽度
  - `1440px` 以下：继续保持左右结构，rail 收缩到更适合桌面端的宽度
  - `1180px` 以下：才允许真正切为单列
- 已保留正确交互方向：
  - 自动滚动只驱动右侧流
  - 点击条目时才联动地图并打开详情
- 已补 UI 冒烟回归：
  - 新增 `1366` / `1440` 桌面宽度
  - 新增首页布局断言，检查地图与 signal rail 是否仍为左右结构

## 验证结果
- `npm run build`：通过
- `node scripts/ui-smoke.mjs`：通过
- 已确认 1366 / 1440 / 1600 / 2880 宽度下首页保持：
  - 左侧地图主舞台
  - 右侧 signal rail
  - 非上下堆叠结构

## 当前结论
- 本轮优先级最高的问题已经纠正：桌面端不会再因为错误断点掉成上下结构。
- 当前首页至少已经从“桌面端骨架错误”恢复到 Crucix 式基础关系：
  - 中央主地图
  - 右侧 signal rail
- 后续再做样式深化时，不得重新引入桌面端单列堆叠规则。
