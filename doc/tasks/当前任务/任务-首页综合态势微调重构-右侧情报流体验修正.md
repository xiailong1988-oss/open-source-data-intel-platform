## 本轮任务背景
用户最新人工测试反馈已经把本轮整改目标收口为三个硬改点：

1. 地图略缩小，给右侧重要情报滚动流让出更明确的空间；
2. 右侧滚动流速度稍微加快，增强持续流动感；
3. 自动滚动时不再联动地图，只有点击条目时才联动地图并打开详情。

本轮不是重新设计首页骨架，而是在当前“地图主区 + 右侧情报流 + 地图顶部窄横条”的结构基础上做定向修正。

## 当前现状承认
- 当前首页已经存在右侧情报流，不是缺少这块内容；
- 当前 `HaidianCockpitPanel.vue` 中已经存在 `activeTickerIndex / currentTicker / focusPoint / selectTickerItem` 等基础逻辑；
- 当前问题集中在三个方面：
  1. 地图与右侧滚动流的宽度比例不合理；
  2. 滚动节奏偏弱；
  3. 自动滚动和地图联动耦合过深，导致地图持续跳动。

## 本轮执行边界
- 只围绕首页驾驶舱相关文件做小范围修正；
- 不重构全站其他页面；
- 不新增新的主结构模块；
- 不恢复底部主轮播框；
- 不重新引入左侧大摘要墙。

## 分阶段执行计划
### 阶段1：主体区比例修正
- 调整地图与右侧滚动流宽度比例；
- 目标：地图仍是主体，但右侧流不再像狭窄附属边栏。

### 阶段2：滚动速度与悬停体验
- 适当加快滚动节奏；
- 如实现允许，补充 hover 暂停、移开继续。

### 阶段3：自动滚动与地图解耦
- 自动滚动只更新右侧流；
- 禁止自动滚动驱动地图聚焦、点位跳动和焦点弹层切换。

### 阶段4：点击条目联动详情
- 点击某条情报时，才执行：
  - 当前项高亮；
  - 地图聚焦；
  - 进入对应详情页。

### 阶段5：文档与回归
- 补充或调整 ticker 假数据；
- 更新当前执行状态；
- 运行构建与 UI 冒烟验证；
- 在文档中记录本轮修正内容和剩余问题。

## 预期验收标准
1. 地图横向占比略缩小，右侧情报流更明显；
2. 滚动速度比当前版本稍快，但仍可读；
3. 自动滚动过程中地图不再持续跳动；
4. 点击右侧情报条目时，地图联动并打开详情；
5. 顶部窄横条仍保持轻量，不再扩厚。

## 本次改动说明
### 阶段1：主体区比例修正
- 已调整首页主体区比例：
  - 地图主区改为 `minmax(0, 2.9fr)`
  - 右侧情报流改为 `minmax(360px, 1fr)`
- 在 `1860px` 以下宽度补了单独比例回退，保证右侧流仍然具备足够存在感；
- 本轮没有改动地图顶部窄横条厚度，保持当前轻量方向。

### 阶段2：滚动速度与悬停体验
- 自动推进间隔从 `5200ms` 调整到 `4200ms`；
- 右侧滚动轨道位移动画从 `900ms` 调整到 `720ms`；
- 新增右侧滚动流 hover 暂停，移出后继续自动推进。

### 阶段3：自动滚动与地图解耦
- 已移除自动滚动内对 `focusPoint(nextItem.relatedPointId)` 的调用；
- 自动滚动现在只负责：
  - 推进 `activeTickerIndex`
  - 更新右侧流内部高亮与视觉位移
- 地图焦点、点位弹层和右侧焦点解释不再被自动滚动强制带着跳。

### 阶段4：点击条目联动详情
- 继续保留“点击右侧情报条目 -> 地图联动 + 打开详情”的主链路；
- `上一条 / 下一条` 控制键本轮也已改为只服务滚动流本身，不再等同于“打开某条详情”；
- 首页因此形成清晰边界：
  - 自动滚动：只滚动流
  - 上一条 / 下一条：只切换流
  - 点击具体条目：联动地图并打开详情

### 阶段5：数据与回归
- 已复核 `src/mock/dashboardCockpit.ts` 中 ticker 数据量；
- 当前默认图层和其他主要图层下的 ticker 数据已足够支撑持续滚动，本轮未额外扩充 mock；
- 已补充 UI 回归脚本对以下行为的验证：
  - 自动滚动会推进右侧流
  - 自动滚动不会带动地图焦点变化
  - hover 可暂停
  - 移出后可恢复
  - 点击条目会跳转详情
