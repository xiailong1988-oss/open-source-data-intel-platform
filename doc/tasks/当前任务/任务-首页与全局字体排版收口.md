# 任务：首页与全局字体排版收口

## 任务状态
- 状态：进行中
- 开始时间：2026-04-02
- 最后更新时间：2026-04-02
- 任务性质：基于最新人工验收，对首页与重点页做结构收口、字号收口、布局机制补全，不再只修局部样式

## 最新问题来源
根据最新人工验收，当前首页和若干重点页仍存在以下问题：
1. 首页模块顶部仍有大量英文标头，占用空间且不符合中文业务系统语境；
2. 首页底部存在留白，首屏有效信息密度仍不够高；
3. 隐藏侧边栏后，地图扩张明显大于其他区块，没有做到整体按比例放大；
4. 右侧重要情报流可视区偏小，滚动感不强；
5. 首页只有左侧局部模块和底部矩阵支持拖拽/resize，主要区块缺少尺寸调节机制；
6. 智能检索、智能问答、预警、报告等重点页仍存在英文 eyebrow、字号偏大和页头偏厚的问题。

## 本轮目标
1. 首页去掉英文标头与冗余眉标，改成更紧凑的中文总览表达；
2. 首页底部留白收口，让首屏展示更多有效信息；
3. 隐藏侧边栏时，首页左中右下四区按比例一起放大，不再只放大地图；
4. 扩大右侧情报流的可视区域，让用户能直接看到持续滚动效果；
5. 为首页主要非地图区块补尺寸调节能力与持久化；
6. 全局收口重点页的字号、页头、英文 eyebrow 和卡片密度。
