## 任务背景
根据最新截图和最新反馈，上一轮首页虽然已经具备：
- 顶部极轻状态条
- 左侧系统脉冲区
- 中央视态地图舞台
- 右侧 signal rail
- 底部摘要矩阵

但仍然存在四类核心问题：
1. 地图高度过高，首屏比例失衡；
2. 左侧系统区描述过多，不够像总览快照；
3. 底部六大板块命名与原始需求不一致；
4. 当前拖拽只支持排序，不支持 resize 与自适应重排。

## 本轮目标
把首页进一步改造成：
- 地图按比例自适应缩小，但仍固定为主舞台；
- 左侧系统区改成纯数字快照；
- 底部六大板块回到原始需求体系；
- 左侧和底部模块支持拖拽 + resize + 布局保存；
- 地图不参与布局拖拽与 resize；
- 首屏完整露出底部六大板块摘要矩阵。

## 2026-04-02 追加整改：首页排版与字号收口
### 新增问题判定
根据最新人工验收，本轮首页虽然已经完成四区结构和布局机制重构，但仍然存在两个新问题：
1. 首页多个区域字号偏大，导致总览页的信息密度不合理；
2. 首页存在“重叠压盖”现象，根因不是单个样式错误，而是多套尺寸机制叠加：
   - 主舞台固定比例高度；
   - 左侧与底部 GridStack 单元高度偏大；
   - 右侧 rail 固定可视行数与固定行高；
   - 地图 overlay strip 与联线层均使用绝对定位。

### 本轮整改目标
1. 收口首页字号层级，统一缩小一档到两档；
2. 收口首页面板密度，减少按钮、标签、状态块的占位；
3. 重排首页四区高度与内部滚动机制，避免首屏内发生压盖；
4. 继续保持现有正确逻辑：
   - 自动滚动不驱动地图；
   - 点击条目时地图联动并打开详情；
   - 左中右下四区结构不回退。

### 本轮计划
1. 先修骨架机制：调整舞台高度、rail 可视行数、GridStack 单元高度；
2. 再统一字号与按钮密度；
3. 再处理地图 overlay、联线层、rail 焦点区的层级与内边距；
4. 最后补 1366 / 1536 / 1600 / 1920 桌面端回归。

### 本次改动说明
1. 首页主舞台高度继续下调，`HaidianCockpitPanel.vue` 中的舞台高度已由更高的 `40vh` 收到 `36vh`，并同步缩小内边距、gap 和地图 overlay 占位；
2. 右侧 rail 不再使用偏大的固定可视区：
   - ticker 行高由 `92px` 调整为 `76px`
   - 可视克隆行数由 `6` 调整为 `5`
   - viewport 改成跟随容器剩余空间，而不是固定高度硬撑；
3. 左侧系统统计区继续保持纯数字，但整体字号和密度下降一档：
   - `GridStack cellHeight` 由 `118` 收到 `96`
   - 主数字由 `24px` 收到 `20px`
   - 按钮、handle、标签间距同步缩小；
4. 底部六大板块矩阵同步收口：
   - `GridStack cellHeight` 由 `138` 收到 `112`
   - 主数字由 `28px` 收到 `22px`
   - highlights、状态标签和内边距统一下调；
5. 顶部 chrome、地图 strip、地图联线层全面收口：
   - 顶部状态条压薄
   - strip 从桌面端双层错误断点恢复为桌面端单层
   - 联线卡片宽度、字号、锚点高度全面下调，减少对地图的压盖；
6. 桌面端错误断点已继续纠正：
   - `SituationMapOverlayBar.vue` 和 `SituationSignalRail.vue` 原先在 `1560px` 就切换为堆叠/双层模式
   - 现已统一改到 `1180px` 以下才真正进入小屏降级。
