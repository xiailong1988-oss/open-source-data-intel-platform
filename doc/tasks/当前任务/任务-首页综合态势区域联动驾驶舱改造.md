## 任务目标

将首页“综合态势”升级为以海淀区地图为核心、支持重点地区切换、支持快讯条与主情报流联动、支持非地图面板拖拽缩放与自适应重排的区域态势驾驶舱首页。

## 本轮改造范围

1. 首页地图主态势区重构
2. 地区切换栏与地图控制区接入
3. 实时情报快讯条新增
4. 主情报流升级为两层结构
5. 左侧系统统计模块区域联动
6. 六大业务板块联动与尺寸调节
7. 首页统一状态管理收口
8. 非地图面板拖拽与缩放能力增强
9. 首页视觉、动效、响应式与自测补齐

## 关键约束

1. 地图必须是首页第一视觉核心，但不能继续占用过高纵向空间。
2. 地图不允许用户自由拖动、滚轮缩放，只允许程序化视角切换。
3. 地区逻辑不得写死在页面结构内，必须抽象为统一配置。
4. 所有首页模块必须统一响应页面级 `currentRegion` / `currentMapLayer` / `layoutState` 等状态。
5. 地图不参与用户拖拽和手动缩放；其他业务面板需支持拖拽、缩放、恢复默认和布局持久化。
6. 本轮改造只聚焦首页综合态势，不破坏其他业务页面结构。

## 分阶段执行计划

### 第一阶段：结构与状态改造

1. 梳理首页现有组件职责与可复用逻辑
2. 建立 `regionConfig`、首页统一状态和联动派生数据
3. 新增首页专用组件：
   - `RegionSwitcher`
   - `LiveTicker`
   - `OsintStreamPanel`
   - `SituationFocusPanel`
   - `ResizablePanelGrid`

### 第二阶段：地图与区域切换

1. 地图舞台改为“程序化视角控制”
2. 接入区域切换栏
3. 支持全域 / 重点地区高亮与平滑切换
4. 点位、边界、焦点、图层随区域联动

### 第三阶段：快讯条与主情报流

1. 新增顶部实时快讯条
2. 情报流升级为主可读信息流
3. 快讯与主流统一响应地区和图层状态
4. 支持自动滚动、暂停和点击联动

### 第四阶段：面板布局增强

1. 非地图面板统一接入拖拽与缩放
2. 模块尺寸变化后自适应重排
3. 六大业务板块继续保留独立布局持久化
4. 地图固定为系统控制区域，不参与手动拖拽和缩放

### 第五阶段：体验打磨与回归

1. 统一首页视觉层级、间距和动效
2. 补齐加载态、空态和滚动体验
3. 执行构建与首页关键链路验证
4. 回写当前执行状态和本任务文档“本次改动说明”

## 已完成实现

1. 首页骨架已重排为：
   - 地区切换栏
   - 地图控制区
   - 地图主态势区
   - 实时情报快讯条
   - 四块联动分析面板
   - 六大业务板块
2. 首页已建立统一状态：
   - `currentRegion`
   - `activeLayer`
   - `currentTickerFilter`
   - `currentStreamFilter`
   - `tickerMode`
   - `streamMode`
   - 二级面板布局持久化状态
3. 地区切换已支持：
   - 海淀区全域
   - 西北旺镇
   - 上地街道
   - 中关村街道
   - 学院路街道
   - 清河街道
4. 地图已改为程序化视角控制：
   - 禁用自由拖拽
   - 禁用滚轮缩放
   - 支持地区高亮边界
   - 支持地区名称地图内标识
   - 支持点位 / 片区点击联动
5. 快讯条已升级为独立承接层：
   - 支持动态 / 静态模式
   - 支持类型过滤
   - 支持 hover 暂停
   - 点击可进入详情
6. 主情报流已升级为可读流：
   - 更大的标题与摘要层级
   - 区域 / 类型 / 状态 / 来源 / 标签字段
   - 跟随地区与类型过滤联动
7. 非地图面板已支持统一拖拽缩放：
   - 区域摘要
   - 主情报流
   - 焦点详情
   - 事件与片区观察
8. 六大业务板块继续保留 GridStack 布局能力，并已接入当前地区上下文联动。
