## 任务背景

在 `Task-01` 完成工程骨架、`Task-02` 完成数据库底座之后，当前首页联调最先需要打通的是地区切换、地图边界和地图图层接口。因此本轮只执行 `Task-03`，围绕 `region` 模块建立第一批真实可调的地区与地图基础接口。

## 本轮执行目标

本轮只执行 `Task-03`，不跨任务扩展：

1. 搭建 `region` 模块基础结构；
2. 开发地区树、地区详情、地区边界、地图图层接口；
3. 保持 DTO / VO 分层清晰；
4. 保持接口返回结构统一；
5. 补充接口文档与联调说明；
6. 完成基础单元测试和真实接口验收。

## 执行约束

1. 当前阶段只做地区与地图基础接口，不提前实现首页聚合和地图点位；
2. 不提前实现快讯条、情报流和情报主表；
3. 不引入缓存、复杂聚合和额外中间件；
4. 所有返回结构统一通过 `ApiResponse` 输出；
5. 每完成一个小任务，都要在对应文档中回写进度和结果。

## 验收方式

本轮已按以下方式完成验收：

1. `mvn test` 通过；
2. `GET /api/regions/tree` 返回树形地区结构；
3. `GET /api/regions/3001` 返回海淀区详情；
4. `GET /api/regions/3001/geo` 返回结构化 GeoJSON；
5. `GET /api/map/layers` 返回图层配置和解析后的 `styleConfig`；
6. `GET /api/map/layers?layerType=focus` 可按类型过滤；
7. `GET /api/regions/9999` 返回统一错误结构；
8. `GET /doc.html` 返回 `200`。

## 风险与下一步计划

1. `biz_region_geo` 当前仍是占位 `FeatureCollection`，真实边界数据需后续通过 Flyway 脚本补齐；
2. 当前只完成地区与地图基础接口，还未实现地图点位、首页概览、快讯条和情报流；
3. 当前本机验证环境为 MySQL `8.4`，Flyway 会输出兼容性提醒，但本项目现有迁移执行成功；
4. 下一轮建议严格进入 `Task-04：情报与事件核心模块开发`，围绕 `biz_intel_item` 搭建 CRUD 和过滤查询能力。
