# 任务：海淀区驾驶舱地图核心深化与在线底图评估

## 任务背景
本任务基于最新一轮人工验证反馈启动。上一轮“首页驾驶舱与全局用户视角优化”虽然已经完成了地图主位、侧栏专注模式和弱信息收敛，但从真实用户视角看，综合态势首页仍没有彻底达到“地图就是第一主角”的状态。

当前核心问题集中在地图本体：
- 抬头仍偏大，继续挤占地图上方空间
- 左侧 `态势焦点 / 专题图层 / 观察方式` 的固定块占位过大
- 高风险点击后详情承接不够直观
- 重点事件预览框与地图点位之间缺少明确连线
- 底图仍偏抽象，缺少更接近真实地图的语义与切换能力
- 重要事件轮播的位置和组织方式还不够理想

因此本轮任务的目标，是继续把首页推进为：

> 以海淀区 GIS 地图为绝对核心、支持在线底图切换、以右侧情报流承接重要事件、点位与预览形成明确空间指向关系的情报驾驶舱。

## 本次任务目标
1. 进一步提升首页地图优先级，让海淀区 GIS 地图再向上、再放大。
2. 压缩首页抬头与地图周边控制区，把非关键信息收纳进扩展动作、滚动区或更紧凑的控制条。
3. 重构 `态势焦点 / 专题图层 / 观察方式` 的组织方式，减少固定占位。
4. 修复“点击高风险但看不到详情承接”的问题。
5. 为重点事件预览框与地图点位之间补齐明确指向关系。
6. 评估并落地更接近真实地图语义的底图方案，并支持底图切换。
7. 重构重点事件轮播区，使其更适合围绕地图右侧形成连续向下的情报流。
8. 重新验证综合态势首页与相关交互：地图点位、详情卡、列表联动、菜单切换、返回逻辑、抽屉 / 弹窗。

## 本次任务纳入的相关储备项
本轮已吸收并覆盖以下储备方向：
1. 海淀区街道 / 园区级更细粒度 GIS 图层的结构预留
2. GIS 热力覆盖、轨迹联动与更细粒度底图服务的评估预留
3. 首页驾驶舱与专题工作台之间的共享 GIS 数据源抽象预留

说明：
- 本轮优先完成首页地图核心与首屏结构问题。
- 更细粒度图层、热力图、轨迹和区域统计仍保留为后续深化项，但本轮已经把底图、地图组件和数据结构预留出来。

## 本次改动说明
### 1. 地图实现从 ECharts 首页专用态势图切换为 Leaflet 首页专用地图舞台
- 新增 `HaidianCockpitMapStage.vue`，只在首页驾驶舱使用 Leaflet。
- 保持全站其他图表和地图仍走现有 ECharts 体系，不推翻已有结构。
- 这样可以在首页拿到更适合在线底图、真实地图交互和点位连线的能力。

### 2. 首页地图进一步上提并放大
- 首页驾驶舱顶部大标题被压缩成轻量 `Haidian GIS` 标识和细状态条。
- 地图画布本体成为驾驶舱首屏绝对主面积，控制区和摘要信息不再继续抢主位。
- `海淀区空间态势总览` 这种大块标题表达已退居到更轻的身份标识层，不再占用首屏大量垂直空间。

### 3. 左侧固定块收纳为紧凑信号条 + 扩展控制面板
- 原本偏大的 `态势焦点 / 专题图层 / 观察方式` 已拆分：
  - 常驻只保留一条紧凑的信号条：`高风险 / 突发 / 热点片区 / 盯防`
  - 图层、筛选、底图切换收纳进地图右上方的紧凑工具按钮
  - 展开后通过滚动控制面板查看，不再长期占大面积空间
- 这一步直接回应了“这些框没必要这么大”的问题。

### 4. 高风险点击后直接承接详情
- 当前点击高风险等信号条后，会立即选中当前筛选范围内优先级最高的点位。
- 地图会同步聚焦该点位，并打开与点位锚定的地图信息框。
- 右侧焦点详情区会同步更新，不再出现“点了但是没看到有效详情”的感受。

### 5. 地图点位与预览框建立明确空间指向关系
- 地图点位改为真实 marker + 锚定 popup 的方式。
- 当前选中对象的地图预览框由 Leaflet popup 提供箭头和锚点指向，预览框与地图点位有清晰连接关系。
- 这一步替代了之前“浮在地图上但不连点”的问题。

### 6. 在线底图与回退策略落地
- 已接入 `OSM 标准` 在线底图。
- 已提供 `驾驶舱暗色` 底图作为本地回退模式。
- 若在线底图加载失败，会自动回退到驾驶舱暗色底图，并给出提示反馈。
- 本轮没有接入 Google 地图，原因是需要 API Key 与更严格的服务条款；当前选择 OSM 是为了保持免费、可演示、可直接集成。

### 7. 重点事件轮播改为地图右侧情报流
- 原先底部横向条带被重构为地图右侧纵向 `重点情报流`。
- 重要事件与风险项沿右侧连续向下展示，点击后会反向聚焦地图点位并更新详情区。
- 这样既保留了轮播承接能力，又不继续压占地图下方的主要空间。

### 8. 数据与类型结构补强
- `dashboardCockpit` 类型新增底图选项与地图边界配置。
- mock 数据补充 `baseMapOptions` 和 `mapBounds`，为后续街道 / 园区级细化图层和共享 GIS 数据层抽象做预留。

## 本轮修复的问题

## 风险点说明
1. OSM 在线底图受网络环境影响，因此本轮保留了 `驾驶舱暗色` 本地回退策略。
2. 当前首页使用 Leaflet 仅限首页驾驶舱，不会影响全站已有 ECharts 图表体系；后续若扩展到更多 GIS 页，需要再评估统一抽象层。
3. `leaflet` 及其类型定义已新增到依赖中，当前构建正常，但后续若继续扩大地图能力，需要关注包体和地图资源加载策略。
