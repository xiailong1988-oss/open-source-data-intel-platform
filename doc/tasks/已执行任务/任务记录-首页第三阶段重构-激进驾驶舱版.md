# 任务：首页第三阶段重构（方案三：激进驾驶舱版）

## 任务状态
- 状态：已完成，待人工验收
- 开始时间：2026-04-01
- 最后更新时间：2026-04-01
- 任务性质：在现有地图主舞台与右侧 signal rail 基础上，升级为“左侧系统脉冲区 + 中央视态地图舞台 + 右侧重要情报流 + 底部六大板块摘要矩阵”的第三阶段驾驶舱首页

## 任务背景
当前首页前两轮已经完成了这些正确方向：
1. 地图仍然是首页主舞台；
2. 右侧重要情报滚动流继续保留；
3. 自动滚动不再驱动地图；
4. 点击条目时才联动地图并打开详情；
5. 顶部维持极轻 chrome，地图顶部维持单条窄 strip。

但最新任务要求已经进入第三阶段，不再满足于“地图 + 右 rail”。
本轮必须补齐：
- 左侧系统脉冲区；
- 底部六大板块摘要矩阵；
- 地图一键显示全部重要信息与联线层；
- 左侧/底部模块拖拽与布局持久化；
- 更明确的现代动效与科技感。

## 本轮目标
把首页升级为：

```text
顶部极轻状态条

左侧系统脉冲区 + 中央视态地图舞台 + 右侧重要情报流

底部六大板块摘要矩阵
```

并满足以下关键体验：
1. 首屏可以完整看到底部摘要矩阵；
2. 地图仍然是中心主舞台，但高度明显收下来了；
3. 左侧显示系统/数据/资源/智能体脉冲；
4. 底部显示六大板块业务摘要；
5. 地图支持一键展开全部重要信息并通过联线指向真实点位；
6. 左侧和底部模块可拖拽换位并保存布局；
7. 页面具备可感知的现代前端动效。

## 本轮实现内容
### 1. 首页布局升级为“左中右下”四区结构
已将首页主骨架升级为：
- 顶部极轻状态条；
- 左侧系统脉冲区；
- 中央视态地图舞台；
- 右侧 signal rail；
- 底部六大板块摘要矩阵。

### 2. 左侧系统脉冲区接入
新增左侧系统脉冲区，并使用系统类指标而不是业务摘要，包含四组模块：
- 采集能力
- 数据存储
- 系统资源
- 智能体与模型

每组模块均具备：
- count-up 数字动效；
- hover 微交互；
- 点击跳转；
- 拖拽排序；
- localStorage 布局持久化。

### 3. 底部六大板块摘要矩阵接入
新增六大板块摘要矩阵，包含：
- 企业主体
- 科技创新
- 企业融资
- 产业资金风险
- 产业空间与园区
- 热点事件与舆情

每个格子均具备：
- 主值 + 增量 + 风险等级；
- 关键摘要；
- 三个高价值子指标；
- 点击跳转；
- 拖拽换位；
- localStorage 布局持久化。

### 4. 地图一键显示全部重要信息
本轮新增地图联线层：
- 支持通过地图顶部 strip 按钮一键展开/收起“全部重要信息”；
- 展开后，重要信息统一吸附在地图顶端或底端；
- 每条信息通过 SVG 联线连接到真实地图点位；
- hover 信息项时联线与点位同步高亮；
- 点击信息项时联动地图并打开详情。

### 5. 地图现代化动效增强
地图舞台本轮补充了：
- 柔和光点光标；
- marker hover 放大；
- 重点点位 pulse；
- 网格与扫描线细节；
- 更弱化的业务容器边框感。

### 6. 右侧 signal rail 继续保留正确方向
本轮继续保留并验证以下正确行为：
- 自动滚动只推动右侧情报流本身；
- 自动滚动不驱动地图焦点；
- hover 可暂停；
- 点击条目时地图才联动，并打开详情页。

## 本轮新增数据结构
### systemMetrics
用于左侧系统脉冲区，覆盖：
- 采集节点
- 在线/离线节点
- 存储总量与今日新增
- CPU / 内存 / 磁盘 / GPU 资源
- 智能体 / 模型 / 推理任务 / 调用量

### businessBoardMetrics
用于底部六大板块摘要矩阵，每个板块包含：
- currentValue
- increment
- trend
- riskLevel
- summary
- highlights
- detailTarget

### mapHighlights
用于地图联线层，包含：
- pointId
- title
- time
- area
- level
- status
- preferredAnchor
- detailTarget
