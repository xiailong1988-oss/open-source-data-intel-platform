# 任务：专题工作台、简报中心与数据健康收尾增强

## 任务背景
当前项目已经完成 GIS 综合态势升级与主要交互整改，平台整体进入收尾深化阶段。根据当前执行状态与后续任务储备，仍需补齐经济专题与创新专题工作台、简报中心正式化改造、信源与数据健康可视化页面，以及 mock -> API 抽象层的准备层，确保项目从“核心工作台已成型”进一步收口到“模块完整、结构稳定、便于后续真实落地”。

## 本次任务目标
1. 落地经济专题与创新专题两个独立工作台页面，并保持与 GIS 首页互跳和全局专题上下文一致。
2. 将报告中心深化为正式化简报中心，补齐简报列表、模板、预览、导出与继续追踪能力。
3. 在数据接入域补齐信源列表与数据健康可视化页面，形成“数据从哪里来、当前是否健康”的可观察能力。
4. 建立 mock -> API 抽象层的基础结构，为后续真实接口接入预留稳定入口。
5. 保持现有菜单、工作台风格、交互闭环和构建稳定性不被破坏。

## 设计与实现原则
1. 不新增与现有结构脱节的大模块，优先接入现有数据接入、报告中心、综合态势等模块体系。
2. 专题工作台必须有清晰的经济/创新视角差异，不能只是换文案。
3. 简报中心必须体现正式输出能力，具备列表、模板、预览、导出和来源回溯。
4. 信源与数据健康页面必须具备可观察性，不做纯静态说明页。
5. API 抽象层以“可平滑替换 mock”为目标，不做过度复杂的真实请求逻辑。
6. 所有新增核心函数、筛选逻辑、路由联动逻辑必须补充中文注释。

## 预期产出
1. 新增经济专题、创新专题、简报中心、信源列表、数据健康页面并可稳定访问。
2. 首页和全局专题视角可跳转到专题工作台，专题页可回流到检索、问答、简报和 GIS 首页。
3. 报告中心与简报中心形成“正式报告 + 快速简报”双输出结构。
4. 数据接入模块形成“数据源管理 + 采集任务 + 信源列表 + 数据健康”完整闭环。
5. 新增 `services` 层承接部分 mock 数据访问，为真实接口接入准备基础结构。
6. 构建与 UI 冒烟回归继续通过，并补充新增页面的基础验证。

## 验收方式
1. `npm run build` 通过。
2. `npm run smoke:ui` 通过，且新增页面可正常打开。
3. 专题工作台、简报中心、信源列表、数据健康页面均具备明确主视图、筛选、联动和空态反馈。
4. 当前执行状态、任务记录和后续任务储备在任务结束后同步更新。

## 本次改动说明

### 专题工作台落地内容
1. 新增经济专题与创新专题工作台路由：`/topic-workbench/economy`、`/topic-workbench/innovation`。
2. 新增 `TopicWorkbenchView.vue`，复用 GIS 空间态势面板，补齐专题指标、事件主线、推荐问题和简报引用区。
3. 新增 `topicWorkbench.ts` mock 和 `topicWorkbench.ts` 类型定义，并基于首页 GIS mock 过滤出经济 / 创新两套专题视图。
4. 首页快捷入口和最近关注已接入专题工作台跳转，形成首页 -> 专题工作台 -> 检索 / 问答 / 简报 / 报告 的闭环。

### 简报中心正式化改造内容
1. 新增 `ReportCenterLayout.vue`，将报告中心调整为“正式报告 / 简报中心”双标签结构。
2. 新增 `BriefingCenterView.vue`，支持四类简报模板、列表筛选、预览、新建生成和 Word / PDF 导出 mock 交互。
3. 重写 `ReportCenterList.vue`，切换为服务层读取，并补充简报中心入口和正式报告指标概览。
4. 重写 `ReportDetailView.vue`，接入服务层并保留 fallback 详情逻辑，继续支持回溯检索、事件、实体和内容。

### 信源与数据健康页面内容
1. 在 `DataAccessLayout.vue` 中新增“信源列表”“数据健康”两个工作台标签。
2. 新增 `SourceIntelligenceView.vue`，展示信源分类、状态、覆盖范围与最近刷新时间，并联动数据源管理和检索。
3. 新增 `DataHealthView.vue`，展示健康分布、新鲜度、异常 / 延迟对象和刷新记录，并联动采集任务页。
4. 新增 `sourceCenter.ts` mock 和类型定义，承接信源目录、健康状态和刷新日志。

### mock -> API 抽象层准备内容
1. 新增 `src/services/mockRequest.ts` 统一模拟请求延迟和深拷贝返回。
2. 新增 `reportCenterService.ts`、`dataAccessService.ts`、`topicWorkbenchService.ts`，作为后续真实接口替换入口。
3. 当前专题工作台、正式报告、简报中心、信源列表和数据健康页面已优先接入服务层。
4. 保留现有 mock 数据组织方式，但页面不再直接依赖新增领域的 mock 文件。

### 路由与交互稳定性收口
1. 新增 `meta.activeMenu` 并调整 `SideMenu.vue`，解决隐藏工作台页面的菜单高亮和切换一致性问题。
2. `scripts/ui-smoke.mjs` 已加入新增页面和关键交互验证，包括专题工作台、简报中心、信源列表和数据健康页面。
3. 菜单链路中的报告中心断言已调整为 `/report-center/reports`，保持新的模块化结构稳定。
4. 本轮未发现新增阻断性的菜单失效、抽屉残留遮罩或按钮无反馈问题。
