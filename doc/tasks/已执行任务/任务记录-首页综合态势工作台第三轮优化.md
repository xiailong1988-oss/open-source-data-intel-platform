# 任务：首页综合态势工作台第三轮优化

## 任务背景
当前首页已经具备深色工作台框架、地图总览、快捷入口和业务联动能力，但整体信息层级仍偏平，首页各区块之间的“主次关系”和“情报工作台节奏”还可以进一步收束，尤其是在大屏宽度下，地图之外的上层摘要和下层追踪区仍有继续强化空间。

## 本次任务目标
1. 强化首页作为整个平台总入口的层级关系，让地图与态势摘要形成更清晰的主线。
2. 优化首页顶部摘要、指标区、辅助分析区和业务联动区的栅格结构。
3. 在不破坏现有地图、钻取、快捷入口与列表联动的前提下，增强首页的大屏信息密度与可读性。
4. 为首页关键数据组装和跳转逻辑补充必要注释。

## 执行范围
### 代码范围
- `src/views/dashboard/OverviewDashboard.vue`
- `src/components/dashboard/SpatialOverviewPanel.vue`
- `src/components/dashboard/DashboardQuickWorkbench.vue`
- 如有必要，可新增首页专用的轻量组件或补充首页 mock/types 字段，但不扩散到其他业务模块。

### 文档范围
- `doc/rules/当前执行状态.md`
- `doc/tasks/当前任务/任务-首页综合态势工作台第三轮优化.md`
- 需要时同步更新 `doc/tasks/待执行任务/后续任务储备.md`

## 设计与实现原则
1. 首页必须继续遵守“地图主视图优先、辅助面板次之、图表与追踪区再其次”的布局逻辑。
2. 不新增大模块，不推翻现有首页地图和钻取链路。
3. 优先通过结构重组、信息分层和面板主次关系优化，而不是单纯加更多盒子。
4. 所有新增的首页核心计算、摘要生成和跳转逻辑都要补注释。
5. 必须兼顾 2880、1920、1600、1440 四类宽度。

## 预期产出
1. 首页顶部具备更强的态势摘要与今日主线表达。
2. 指标卡区、工作台入口区、地图区、分析区和业务追踪区形成更稳定的视觉层级。
3. 首页在大屏宽度下更适合作为演示和日常值守入口。
4. 构建验证通过，且文档闭环完整。

## 验收方式
1. `npm run build` 通过。
2. 首页路由、地图、快捷入口、指标卡钻取、图表钻取和底部列表跳转保持可用。
3. 首页结构相较上一轮有明显层级提升，但未破坏原有联动。
4. 当前任务文档与执行状态在任务完成后同步更新。

## 本次改动说明

### 一、页面整改范围
本次重点整改了以下首页相关文件：
- `src/views/dashboard/OverviewDashboard.vue`
- `src/components/dashboard/DashboardSituationDigest.vue`
- `src/components/dashboard/SpatialOverviewPanel.vue`

### 二、首页层级优化动作
1. 新增首页态势摘要组件
   - 将值守摘要、今日主线、重点区域从分散信息中抽离出来
   - 形成首页第一屏右侧高价值摘要区
   - 支持直接跳转到预警、任务、专题、报告与区域检索
2. 重组首页顶部结构
   - 首页由“英雄区 + 指标区 + 右侧态势摘要区”组成第一屏
   - 指标卡从 6 列缩为 3 列主栅格，增强单卡可读性与留白
3. 强化辅助分析区层级
   - 趋势图改为主分析位，热点排行作为高权重长条图块
   - 来源分布和类型分布降为辅助分析位
4. 强化业务联动区层级
   - 最新预警升级为主列表位
   - 热门专题改为高权重纵向专题面板
   - 最近采集任务和最近报告保留为下层追踪位
5. 强化地图卡片信息组织
   - 增加高风险点位统计
   - 增加“今日重点区域”焦点条，可快速切换区域观察

### 三、关键稳定性处理
- 保持首页原有指标卡钻取逻辑不变，只在结构层重排
- 保持图表 click drill-down 与列表跳转逻辑不变
- 保持地图与右侧联动面板的已有跳转能力不变
- 对摘要区新增点击项统一做路由收口，避免在模板里分散跳转实现

- 地图重点区域焦点条与增强统计逻辑
