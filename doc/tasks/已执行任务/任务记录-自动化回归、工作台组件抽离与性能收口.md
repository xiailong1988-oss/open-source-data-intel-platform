# 任务：自动化回归、工作台组件抽离与性能收口

## 任务背景
当前项目已经完成大部分页面的统一工作台迁移，但仍存在三类共性问题：
1. 重点页面仍缺少可重复执行的大屏 / 常规宽度自动化冒烟与截图回归能力。
2. 首页、事件中心、告警中心等页面存在重复的工作台摘要结构，仍有进一步抽取共用组件的空间。
3. 当前构建包体偏大，尚未完成按路由或模块的拆包收口。

## 本次任务目标
1. 为重点页面补充可执行的 UI 冒烟与截图回归脚本。
2. 抽离首页、事件中心、告警中心可复用的工作台摘要/指标组件，并接入现有页面。
3. 完成按路由或模块的拆包优化，收口当前构建包体问题。
4. 保持现有页面路由、菜单、筛选、详情和钻取交互稳定。

## 设计与实现原则
1. 自动化脚本必须以当前项目本地运行方式为基础，不引入收费或重型平台依赖。
2. 共用组件抽离必须优先解决真实重复结构，不能为了抽象而抽象。
3. 拆包优化必须以不破坏现有路由与页面加载为前提。
4. 新增脚本、组件和关键拆包逻辑必须补注释，说明用途与边界。
5. 所有改动完成后必须至少验证：构建通过、脚本可运行、关键页面可正常渲染。

## 预期产出
1. 项目具备基础的 UI 冒烟与截图回归脚本，可覆盖首页、检索、问答、预警、报告等重点页面。
2. 项目新增至少一组高复用的工作台摘要/指标组件，并在首页、事件中心或告警中心接入。
3. 构建输出完成拆包优化，尽量消除或显著缓解当前包体过大的问题。

## 验收方式
1. `npm run build` 通过。
2. 自动化冒烟 / 截图脚本可执行并产出结果。
3. 首页、事件中心、预警中心等接入新组件后交互正常。
4. 当前任务文档、执行状态和后续任务储备在任务结束后同步更新。

## 本次改动说明

### 改动说明
1. 新增 `scripts/ui-smoke.mjs` 与 `npm run smoke:ui`，自动拉起本地服务并对首页、智能检索、智能问答、预警概览、报告中心进行 1600 / 2880 两组宽度截图回归，同时验证关键菜单链路。
2. 新增 `WorkbenchHero.vue` 与 `WorkbenchMetricGrid.vue`，把工作台首屏摘要与概览指标抽成共用组件，并接入首页、事件中心、预警中心以及最近迁移过的历史工作台页面。
3. 将路由改为懒加载，新增 `src/lib/echarts.ts` 统一注册图表核心模块，减少整包图表引入带来的构建体积压力。
4. 在 `vite.config.ts` 中补充拆包策略，对路由级 chunk、图表 vendor 和 UI vendor 做进一步收口，并把构建 warning 控制到当前可接受范围。

### 新增组件说明
1. `WorkbenchHero.vue`：统一工作台首屏 Hero 结构，承接标题、说明与右侧 meta 区。
2. `WorkbenchMetricGrid.vue`：统一工作台概览指标栅格，支持可点击指标与静态指标混排。
3. `src/lib/echarts.ts`：统一图表核心模块注册，不直接作为 UI 组件使用，但作为图表能力的公共入口。

### 涉及接口说明
- 本轮未新增真实后端接口。
- 新增的 UI 冒烟脚本仅依赖本地 `vite` 开发服务，不接外部平台。
- 图表能力仍基于现有 mock 数据和 ECharts 前端渲染，不接真实 GIS 或后端统计服务。

### 风险点说明
1. `Element Plus` 仍然是当前最大 vendor 依赖块，虽然构建已收口，但若后续要继续降体积，仍建议评估更细粒度按需引入。
2. UI 冒烟脚本当前覆盖重点页面与菜单链路，后续若新增核心模块，应同步扩展脚本路由清单。
3. 当前截图回归以首屏视角为主，后续如需覆盖更深层交互，可继续补登录态、弹窗态和详情态截图。
