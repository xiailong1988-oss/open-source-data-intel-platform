# 待办：首页驾驶舱与全局用户视角优化

## 任务背景
本待办基于最新一轮人工试用反馈整理，目标不是继续堆内容，而是把“用户第一眼先看到什么、先理解什么、先点击什么”重新排清楚。当前项目已经具备较完整的工作台能力，但在首页驾驶舱和全局首屏体验上，仍然存在明显的认知优先级问题。

## 已识别的问题
### P0：首页首屏认知问题
- GIS 地图虽然已经存在，但首屏仍会被大标题、摘要块和次级状态分散注意力。
- 首页大标题和说明区体量偏大，挤占地图之前的垂直空间。
- `高风险 / 突发事件 / 热点片区 / 重点关注` 原先占位过大，信息密度偏低。
- 图层切换与筛选主要堆在地图上方，像普通工具栏而不是围绕地图服务。
- `当前值守 / 最新刷新` 等次级状态覆盖主视图，影响地图的第一视觉优先级。
- 地图缺少“地图之外的高优先级信息承接带”，当点位较多时用户仍然容易漏掉重点。

### P1：全局布局与工作台密度问题
- 左侧菜单虽然支持折叠，但还不够像“为驾驶舱释放画布”的专注模式。
- 顶栏信息密度偏高，首页首屏容易被系统状态、说明和次级入口分散注意力。
- 多个工作台页仍保留“先指标、后重点”的默认顺序。
- 部分右侧焦点卡和辅助观察卡在大屏下体量偏大。

### P2：可读性与回归问题
- 深色背景下个别次级文字、hint 和弱说明的对比度不够。
- 一些 support card 的说明文本长度偏长。
- 自动化回归过于依赖按钮和标题文本，结构稍改就容易误报或卡住。

## 本轮已完成的改造
- [x] 将首页首屏调整为地图优先，GIS 主画布先于摘要和统计卡出现。
- [x] 压缩首页大标题与说明区，改为地图卡片内轻量标识。
- [x] 将四类重点态势改为紧凑态势条，围绕地图左侧组织。
- [x] 将图层切换、显示模式改造为地图左侧 rail 控件，而不是继续堆在地图上方。
- [x] 下沉并弱化次级系统状态，避免覆盖主视图。
- [x] 新增重点信息轮播条带，形成“地图 + 条带”的双通道承接。
- [x] 为侧栏补齐隐藏 / 展开操作，并在首页与专题工作台启用更宽的沉浸式画布。
- [x] 收紧顶栏、工作台 Hero 区和指标卡密度，降低非关键信息体量。
- [x] 提升深色主题下次级文字与 hint 的可读性。
- [x] 重构 UI 冒烟脚本，使其基于稳定结构选择器执行验证。

## 仍建议后续继续跟进的事项
- [ ] 海淀区街道 / 园区级更细粒度 GIS 图层。
- [ ] GIS 热力图、轨迹联动、区域统计等高级态势表达。
- [ ] Element Plus 更细粒度按需引入与进一步性能收口。
- [ ] 截图基线比对与视觉差异告警体系。

## 结论
这份待办的核心判断已经落地验证：后续整改的重点不再是“页面有没有内容”，而是“人类用户第一眼能不能先抓住最重要的信息”。本轮首页和全局首屏结构已经按这一原则完成第一阶段收口，剩余事项属于深化增强，而不是阻断问题。
