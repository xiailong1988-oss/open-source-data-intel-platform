## 三、综合态势页面的重构要求

### 总体思路
你需要站在人类真实用户的角度思考：

> 用户进入综合态势页面后，如何在最短时间内，一眼看到最多、最重要的信息，同时页面又不显得乱？

所以页面必须做到：
- 信息主次分明
- 核心区域突出
- 高频信息优先展示
- 次级信息有序收纳
- 视觉上有节奏感
- 不堆砌、不杂乱、不平均用力

### 核心原则
综合态势页面必须从“模块拼盘”调整为“以 GIS 地图为中心的信息工作台”。

也就是说：
- 地图是主角
- 预警、分布、重点区域、专题切换、详情面板都是围绕地图服务
- 用户第一眼应先看到态势全貌，再逐步查看细节

## 四、GIS态势图的具体要求

### 1. 地图定位
- 使用中国北京作为核心底图
- 地图要有真实 GIS 感，而不是简单示意图
- 可以结合现有技术栈，选择合适实现方式（如 WebGIS 地图组件、地图底图服务、行政区轮廓、热点区域覆盖等）
- 视觉上要体现“城市态势分析”的感觉

### 2. 地图要成为页面视觉中心
在综合态势页面中：
- GIS地图必须占据最重要、最大、最核心的位置
- 不能再让地图缩在角落或和普通卡片并列弱化
- 用户打开页面第一眼必须先看到地图态势区

### 3. 图层化展示
围绕 GIS 地图设计图层系统，例如可以包括但不限于：
- 风险预警图层
- 热点区域图层
- 重点事件图层
- 专题任务图层
- 关注目标分布图层
- 统计分析覆盖图层

要求：
- 支持图层开关
- 不同图层之间视觉表达有区分
- 图层切换后，相关侧边信息和底部信息可以联动变化
- 图层不是摆设，要有实际信息组织作用

### 4. 地图联动机制
地图不只是展示，要成为联动核心。至少应考虑这些联动：
- 点击区域 -> 更新右侧详情面板
- 点击风险点 -> 展示对应预警信息
- 切换图层 -> 更新地图覆盖内容 + 侧边摘要
- 点击重点事件 -> 定位到地图对象或高亮相关区域
- 从列表点击某条记录 -> 地图同步聚焦/高亮

### 5. 地图信息密度控制
要解决“既让用户看到很多重要信息，又不显得乱”的问题，因此必须注意：
- 信息要分层展示
- 默认展示最关键内容
- 次级内容通过图层、筛选、展开、抽屉等方式承载
- 不要把所有东西一次性全部堆在地图上
- 对点位、区域、热区、告警、事件使用清晰的分级展示策略

## 五、综合态势页面的信息结构建议（必须重做，不要沿用现在的平铺思路）

你需要重新组织页面的信息层级，建议方向如下：

### 1. 顶层：关键态势总览
可放在地图上方或与地图强关联的位置，展示最关键的概览指标，例如：
- 今日预警数量
- 高风险区域数量
- 重点关注事件数量
- 正在跟踪专题数量
- 风险趋势变化

要求：
- 精简
- 强调关键信息
- 不能喧宾夺主压过地图

### 2. 中心层：GIS地图主工作区
这是页面绝对核心。
应包含：
- 北京底图
- 图层控制
- 区域高亮
- 风险点位/热区展示
- 快速切换专题或预警模式

### 3. 右侧或侧边：联动情报面板
用于承接地图交互后的详细信息，例如：
- 当前选中区域概览
- 当前选中风险点详情
- 当前图层摘要
- 当前重点事件清单
- 当前专题说明

要求：
- 与地图形成主辅关系
- 不能抢地图视觉中心
- 信息结构要清晰，便于快速浏览

### 4. 底部或下方：趋势与列表区
承接需要横向展开的信息，例如：
- 风险趋势图
- 事件时间线
- 重点事件列表
- 预警处置建议
- 区域对比表格

要求：
- 与地图形成上下分区
- 结构稳定
- 支持与地图联动

## 六、用户体验目标

本次页面改造要满足以下体验目标：

### 1. 第一眼就知道重点在哪
用户一进入综合态势页面，就能立刻看出：
- 核心态势在哪里看
- 风险从哪里看
- 地图是主要工作区
- 重点信息在哪里展开

### 2. 重要信息多，但不乱
页面要让人感觉“信息丰富但有秩序”，而不是：
- 模块很多但主次不清
- 各个面板都在抢注意力
- 信息很多却抓不住重点

### 3. 更像分析工作台，而不是普通后台首页
要有“看态势、查风险、点区域、看联动、切图层、做研判”的工作流感。
