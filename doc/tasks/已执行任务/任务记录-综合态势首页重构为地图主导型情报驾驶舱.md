# 任务：综合态势首页重构为地图主导型情报驾驶舱

## 任务背景
当前项目虽然已经完成 GIS 视图、专题工作台、预警中心与报告中心等能力，但综合态势首页仍保留较强的“传统后台看板”痕迹：首页第一视觉仍被统计卡片和常规模块占据，地图虽然存在，却还不是页面绝对主位，也没有真正围绕海淀区形成情报驾驶舱式的信息结构。这与当前项目“以 GIS 为核心的情报工作台”定位不一致，因此需要进行一次结构性重构，而不是局部修补。

## 本次任务目标
1. 将综合态势首页重构为“地图优先”的驾驶舱结构，进入首页第一眼必须先看到地图。
2. 将地图聚焦到北京市海淀区，不再泛化到其他区作为首页主范围。
3. 围绕海淀区地图组织风险预警、突发事件、热点事件、重点关注四类图层与相应信息结构。
4. 补齐地图专题切换、点位点击、信息框展示、详情跳转、地图与列表联动。
5. 解决点位较多时的信息降噪问题，避免地图一多就乱。
6. 重新验证首页和主要页面的点击、联动、菜单、抽屉、详情跳转与返回逻辑。
7. 完成后同步更新当前执行状态、界面规范和任务记录说明。

## 设计与实现原则
1. 不做“把地图挪上来”的表层改动，而是重构首页信息主次关系。
2. 地图必须成为首页第一视觉中心，统计卡片只保留为第二层摘要信息。
3. 首页地图必须明确聚焦海淀区，不能继续延用北京全域概览逻辑。
4. 地图周边信息必须围绕地图组织，不能做成地图与若干普通卡片等权并列。
5. 点位较多时必须默认降噪，只显式展示高优先级标注，其余点位以静默符号保留。
6. 所有新增的地图交互、跳转和筛选逻辑必须补中文注释。
7. 不能破坏原专题工作台、预警、问答、报告和数据接入等既有稳定链路。

## 预期产出
1. 首页第一屏调整为海淀区 GIS 驾驶舱，地图占据绝对主位。
2. 新增首页专用的海淀区驾驶舱 mock 数据与地图组件。
3. 首页图层支持：风险预警、突发事件、热点事件、重点关注。
4. 首页地图支持：片区点击、点位点击、信息框、详情跳转、地图与列表联动。
5. 首页默认启用降噪标注策略，避免高密度点位时信息失控。
6. 自动化回归脚本覆盖首页新交互，并验证详情跳转和返回逻辑。

## 本次改动说明

### 首页结构重构内容
1. 删除首页首屏的传统 Hero + 顶部统计卡片优先结构，改为首页先进入 `HaidianCockpitPanel`。
2. 将统计类信息降级为地图后的摘要带，不再占据首屏主位。
3. 将快捷入口、态势摘要和辅助分析图谱整体后移，形成“地图主位 -> 支撑入口 -> 辅助分析”的三层结构。

### 海淀区 GIS 驾驶舱内容
1. 新增 `public/maps/haidian.json` 作为海淀区专用底图。
2. 新增 `src/components/dashboard/HaidianCockpitPanel.vue`，专门承接首页地图主导型驾驶舱。
3. 驾驶舱支持四类图层：风险预警、突发事件、热点事件、重点关注。
4. 驾驶舱支持时间、片区、风险等级、显示模式四类筛选能力。
5. 驾驶舱内置点位降噪逻辑：默认只展开高优先级标注，其余点位以静默符号保留。
6. 驾驶舱支持片区列表、事件快报、专题建议与地图联动。
7. 驾驶舱支持点位信息框、右侧联动情报面板、继续跳转到预警/事件/实体/报告/检索页面。

### 数据与结构稳定性处理
1. 新增 `src/mock/dashboardCockpit.ts` 和 `src/types/dashboardCockpit.ts`，专门服务首页驾驶舱。
2. 保留原 `dashboardMockData.spatialOverview` 继续给专题工作台使用，避免首页改造把专题页带坏。
3. 旧 query 回流到首页时，统一映射为新的驾驶舱图层，避免返回首页后落到错误层级。

### 自动化回归与交互验证
1. 更新 `scripts/ui-smoke.mjs`，将首页 heading 改为“海淀区 GIS 情报驾驶舱”。

3. 保留并继续验证智能检索、智能问答、数据接入、预警概览、预警记录、报告中心、简报中心等页面的回归路径。
4. 继续验证菜单链路：智能检索 -> 综合态势 -> 数据接入 -> 智能问答 -> 报告中心。
