# 任务：剩余历史页面统一工作台迁移

## 任务背景
首页、检索、问答、预警中心、事件中心等重点页面已经进入统一的深色工作台语言，但数据接入和系统管理中仍有部分历史页面保持传统“卡片 + 表格/表单”形态，整体一致性仍不够，尤其是在系统管理区域与数据接入任务页，工作台主辅层级和信息摘要仍然偏弱。

## 本次任务目标
1. 将采集任务管理页迁移到统一工作台形态。
2. 将用户管理页迁移到统一工作台形态。
3. 将角色权限管理页迁移到统一工作台形态。
4. 将系统配置页迁移到统一工作台形态。
5. 不破坏现有表格、表单、弹窗、抽屉、启停、保存等既有业务链路。

## 设计与实现原则
1. 保持现有深色工作台语言，不推翻全局布局和模块页签结构。
2. 每个页面至少形成“概览指标 / 主工作区 / 辅助信息区 / 详情或弹层”的清晰结构。
3. 优先复用现有 `PageSectionHeader`、全局卡片规范和布局 token。
4. 所有新增的关键筛选逻辑、状态切换逻辑、弹层初始化逻辑必须补注释。
5. 必须兼顾 2880、1920、1600、1440 四类宽度。

## 预期产出
1. 采集任务页具备任务概览、队列区、焦点日志和运行观察。
2. 用户管理页具备用户概览、列表区、焦点用户和组织观察。
3. 角色权限页具备角色概览、角色焦点、权限树和配置入口的更清晰主辅层级。
4. 系统配置页具备配置摘要、主表单区和右侧预览/说明区的更清晰层级。

## 验收方式
1. `npm run build` 通过。
2. 数据接入与系统管理相关路由正常打开。
3. 原有新增、编辑、启停、保存、弹窗/抽屉等交互保持可用。
4. 当前任务文档与执行状态在任务结束后同步更新。

## 本次改动说明

### 改动说明
1. 将 `CollectionTaskManagement.vue` 从传统单卡片表格页迁移为“概览指标 + 主队列区 + 右侧焦点日志/类型观察/待关注任务”的工作台结构。
2. 将 `UserManagement.vue` 迁移为“概览指标 + 用户列表 + 当前用户焦点 + 角色/部门观察”的工作台结构，并保留新增、编辑、启停交互。
3. 将 `RolePermissionManagement.vue` 迁移为“概览指标 + 角色列表 + 权限树主区 + 角色焦点/模块观察/最近更新”的工作台结构，并保留权限分配弹窗。
4. 将 `SystemConfigView.vue` 迁移为“配置摘要 + 主表单区 + 右侧预览/说明区”的工作台结构，并保留恢复默认和保存 mock 配置。
5. 同步补充了筛选重置、焦点对象同步、query 接管和配置重置等关键注释，保证后续维护可读性。
