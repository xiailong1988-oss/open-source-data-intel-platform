# 任务：首页驾驶舱与全局用户视角优化

## 任务背景
本任务基于最新一轮人工试用反馈启动。首页虽然已经完成海淀区 GIS 驾驶舱改造，但仍然存在明显的“机器能看到很多、人第一眼却抓不住重点”的问题：地图还没有成为无争议的首屏主位，大标题说明区、态势摘要块、图层控制和系统状态信息仍在抢视线。

同时，这种问题并不只存在于首页，还体现在左侧菜单占位、顶栏信息密度、工作台 Hero 区体量和部分页面“先指标、后重点”的默认顺序上。因此本轮任务不是样式修补，而是一次以“人类首屏认知优先”为原则的结构性优化。

## 本次任务目标
1. 进一步重构综合态势首页，让海淀区 GIS 地图成为无争议的第一视觉中心。
2. 压缩首页首屏中一切会抢地图视线的元素，包括大标题、大摘要块、覆盖型状态 HUD 等。
3. 将高风险点位、突发事件、热点片区、重点盯防改为更紧凑、更高亮、更围绕地图组织的信息条带。
4. 将图层切换、筛选和重点事件承接方式改为“围绕地图服务”，而不是继续在地图上方堆工具条。
5. 为地图补充重要信息轮播 / 条带承接，解决地图无法同时承载所有高优先级信息的问题。
6. 将左侧菜单增强为更明确的折叠 / 隐藏式工作台模式，为驾驶舱释放更大画布。
7. 全局收敛顶栏和工作台页 Hero 区的信息密度，避免非关键内容抢首屏注意力。
8. 对首页和重点页面重新做点击、联动、菜单、弹窗、抽屉、返回逻辑回归验证。

## 设计与实现原则
1. 首页必须遵循“先看地图，再看高优先级态势，再看支撑信息”的顺序。
2. 不允许通过仅调颜色、调字号或简单换位置来冒充结构性优化。
3. 地图周边的所有信息都要服务于地图，不允许再与地图等权并列抢主位。
4. 系统状态类信息必须降级，不允许覆盖地图主视图。
5. 重要信息可以高亮，但不能通过遮挡地图来实现强调。
6. 左侧菜单和顶栏的改动必须以不破坏既有路由和菜单逻辑为前提。
7. 全局工作台页的收敛以“减少无效说明、压缩弱信息、突出主视图”为原则。
8. 所有新增或重构的关键逻辑必须补中文注释。

## 预期产出
1. 首页首屏形成“地图主舞台 + 紧凑侧边控制 + 重要信息条带 + 右侧焦点情报”的结构。
2. 首页大标题区缩小或退居次级，不再抢占地图之前的位置。
3. 首页高风险点位、突发事件、热点片区、重点盯防不再以大卡片形式占用大面积空间。
4. 左侧菜单支持更明显的隐藏 / 收起工作模式。
5. 顶栏和工作台 Hero 区整体密度下降，主内容更突出。
6. 自动化回归脚本补到新版首页结构和关键交互链路。

## 验收方式
1. `npm run build` 通过。
2. `npm run smoke:ui` 通过。
3. 首页进入后第一眼先看到海淀区 GIS 主画布，而不是标题、大摘要块或统计卡。
4. 地图周边信息明显更轻量，重要信息仍能快速进入详情。
5. 左侧菜单收起 / 隐藏后，首页内容区宽度明显增大。
6. 菜单切换、详情抽屉、地图联动、列表跳转、返回逻辑不被破坏。

## 本次改动说明
### 1. 首页结构重构
- 将首页首屏调整为“驾驶舱地图主位 + 下沉支撑摘要”的顺序，`HaidianCockpitPanel` 保持在首页第一屏绝对主位。
- 将首页原本位于地图之前的大说明区彻底压缩，避免用户进入首页先读文案、后找地图。
- 将 `DashboardSituationDigest` 和统计摘要整体后移，让地图先承担“发生了什么、在哪发生”的判断任务。

### 2. 地图周边信息重排
- 将 `高风险 / 突发事件 / 热点片区 / 重点关注` 改为地图左侧紧凑态势条，不再用大卡片分割首屏。
- 将图层切换和显示模式重排到地图左侧 rail 区，形成“围绕地图服务”的控制方式。
- 移除了地图上方会挡住主画布的大片状态 HUD，改为右上角轻量徽标和底部小型信息框。
- 新增“重点信息轮播”条带，用于承接地图放不下的高优先级信息，并与地图焦点和右侧情报面板联动。

### 3. 全局布局与首屏认知优化
- 在 `app` store 中新增侧栏隐藏状态，并在 `MainLayout` 中为首页与专题工作台启用更宽的沉浸式内容区。
- 在 `TopBar` 中增加“隐藏侧栏 / 展开侧栏”操作，并对首页 / 专题工作台启用更轻量的 focus 顶栏样式。
- 调整全局 spacing、工作台 Hero 区、指标卡和 hint 文本，降低非关键信息的体量与侵入感。

### 4. 可读性修复
- 提升了深色主题下的次级文字、提示文字、说明文案与指标 hint 的亮度。
- 收紧首页和工作台的长描述，让弱信息退回次级层，不再抢主视图注意力。

### 5. 自动化验证补强
- 重构 `scripts/ui-smoke.mjs`，不再依赖脆弱的中文标题文本来定位页面和按钮，而是改为基于稳定结构选择器验证。
- 支持通过 `SMOKE_BASE_URL` 与 `SMOKE_SKIP_SERVER` 直接复用当前开发服务执行回归，避免重复起服务导致误判。
- 已将首页驾驶舱的新结构（专注视图、图层切换、轮播联动、详情跳转）纳入回归路径。

## 本轮修复的问题
1. 修复 GIS 地图虽然存在，但第一眼仍不够主位的问题。
2. 修复首页大标题和说明区体量过大、挤占地图首屏空间的问题。
3. 修复态势摘要块占位过大、信息密度偏低的问题。
4. 修复图层切换主要堆在地图上方、像普通 tabs 工具栏的问题。
5. 修复 `当前值守 / 最新刷新` 这类次级状态覆盖主视图的问题。
6. 修复首页缺少“地图之外的高优先级信息承接区”的问题，补齐轮播条带。
7. 修复深色主题下个别弱文字对比度不够的问题。
8. 修复 UI 冒烟脚本在新版结构下容易卡住的问题。
