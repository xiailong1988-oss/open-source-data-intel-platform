# 任务：综合态势首页在线地图替换与边界播放栏整改

## 任务状态
- 状态：已完成，待人工验收
- 开始时间：2026-03-27
- 最后更新时间：2026-03-27
- 任务性质：基于现有首页驾驶舱结构的定向整改，不推翻全站地图与 ECharts 体系

## 任务背景
用户在最新人工测试中明确指出，当前首页虽然已经完成“地图主位化”，但仍存在三个关键偏差：
1. 默认主底图仍不符合国内在线地图使用预期，需要从 `OSM 标准` 升级为“高德在线地图优先”；
2. 首页地图仍存在海淀边界附近的硬限制，放大后无法自然查看边缘与周边空间；
3. 右侧“重点情报流”不能替代底部播放轴，必须恢复地图下缘可见、可操作、可联动的底部事件播放栏。

本轮以用户最新人工反馈为最高优先级。即使旧任务中曾认为某些问题“已完成”，本轮也以最新体验反馈重新判断和整改。

## 参考站学习结论
本轮重新参考以下两个站点的信息组织方式：
- [World Monitor](https://www.worldmonitor.app/)
- [Crucix](https://www.crucix.live/)

### 学到的重点
1. 地图必须是第一视觉中心，控制项应围绕地图收纳，而不是长期压住地图主体。
2. 地图下缘的滚动事件条/播放轴，是承接空间态势变化的重要结构，不能只用右侧列表代替。
3. 左右两侧信息面板应该承担“补充说明”和“对象承接”，而不是抢占地图主舞台。
4. 地图的真实感不仅来自配色，更来自在线底图、边界层级、点位分层和联动反馈的一致性。

### 本项目的落地方式
1. 首页继续保留 `OverviewDashboard + HaidianCockpitPanel + HaidianCockpitMapStage` 的结构，不推翻外层工作台。
2. 首页地图 provider 改为工厂驱动，优先使用高德，保留百度预留位，OSM 降级备用。
3. 底部播放栏恢复为首页首屏的一部分，右侧情报流继续保留，但职责明确为补充情报流。
4. 保留当前首页的右侧详情承接和跳转逻辑，只在地图 provider、边界视野和底部时间轴上做最小必要重构。

## 当前现状识别
本轮不是从零设计首页，而是在以下现有实现上纠偏：
- 首页地图主组件已经拆成 `HaidianCockpitPanel + HaidianCockpitMapStage`；
- 当前底图能力已有 `OSM 标准` 与 `驾驶舱暗色` 回退；
- 当前底部横向事件条已经被右侧纵向“重点情报流”替代；
- 当前地图视野存在海淀范围附近的硬限制；
- 当前地图已经具备点位点击、右侧详情、详情跳转等基础联动能力。

## 本轮改动说明

### 1. 首页底图改为高德在线地图优先
本轮新增了首页专用地图 provider 抽象层：
- `src/lib/map/types.ts`
- `src/lib/map/mapProviderFactory.ts`

当前 provider 结构如下：
- `高德标准`：首页默认主底图
- `高德深色`：首页深色在线底图
- `OSM 标准`：仅保留为开发/降级备用
- `驾驶舱暗色`：本地回退底图
- `百度地图`：仅做 provider 预留，本轮未启用主实现

环境变量约定：
- `VITE_AMAP_KEY`
- `VITE_AMAP_SECURITY_CODE`
- `VITE_BMAP_AK`

相关文件：
- `src/lib/map/mapProviderFactory.ts`
- `src/lib/map/types.ts`
- `.env.example`
- `src/vite-env.d.ts`
- `src/types/dashboardCockpit.ts`
- `src/mock/dashboardCockpit.ts`

### 2. 修复海淀边界硬限制
本轮对 `HaidianCockpitMapStage.vue` 做了以下调整：
- 去掉了首页地图的 `maxBounds` 约束思路，不再把用户拖拽与缩放永久锁死在海淀边框内；
- 将“默认聚焦海淀”与“永久限制用户视野”彻底分开；
- 首页初始化时仅做一次海淀聚焦；
- 新增显式“聚焦海淀”按钮，由用户主动拉回默认视角；
- 放宽初始 `fitBounds / flyToBounds` 的 padding 和默认缩放，让海淀边缘与周边空间显示更自然；
- 去掉旧实现里过于紧边的 `pad(-0.08)` 思路。

相关文件：
- `src/components/dashboard/HaidianCockpitMapStage.vue`
- `src/components/dashboard/HaidianCockpitPanel.vue`

### 3. 恢复底部滚动事件播放栏
本轮明确恢复了地图下缘的底部播放栏，并且不再把右侧情报流解释成播放轴替代品。

当前职责划分：
- 底部播放栏：承担当前图层事件的时间承接、自动播放、上一条/下一条、手动点选与地图联动
- 右侧重点情报流：承担补充情报流、焦点对象摘要、详情承接与快捷跳转

底部播放栏能力：
- 自动播放
- 暂停
- 上一条
- 下一条
- 按图层快速切换
- 点击条目后聚焦对应地图点位
- 同步右侧焦点详情

相关文件：
- `src/components/dashboard/HaidianCockpitPanel.vue`

### 4. 解决播放栏加入后地图 overlay 定位冲突
为了避免底部播放栏与地图浮层互相压住，本轮新增了地图表面容器：
- `haidian-cockpit__map-surface`

现在地图浮层（信号条、工具按钮、控制面板、地图说明）统一相对地图表面定位；底部播放栏作为地图舞台的独立下缘承接区，不再被 overlay 坐标系统误伤。

### 5. 修复 Leaflet 快速切页报错
在回归测试里发现：首页地图快速切页时，Leaflet 在动画未结束时卸载，可能抛出：
- `Cannot read properties of undefined (reading '_leaflet_pos')`

本轮已修复：
- 禁用首页地图上的缩放/淡入类动画
- 组件卸载时显式 `stop()`、`off()`、`remove()`
- 避免地图还在动画时就被路由卸载

相关文件：
- `src/components/dashboard/HaidianCockpitMapStage.vue`

### 6. 更新自动化回归脚本
本轮同步修正了首页改为在线底图后的回归脚本假设：
- `networkidle` 改为 `domcontentloaded`，避免在线瓦片长期请求导致脚本假性卡住；
- 首页交互新增了聚焦海淀、底图切换、底部播放栏点击等校验；
- 专题工作台加入了加载态脱离后的再交互等待；
- 菜单链路改为稳定的索引点击，降低文本编码和渲染差异带来的脆弱性。

相关文件：
- `scripts/ui-smoke.mjs`

## 人工验收建议
建议重点人工验证以下内容：
1. 首页默认底图是否优先进入高德在线方案；
2. 放大海淀地图后，边缘和周边空间是否不再被裁死；
3. 首页底部播放栏是否首屏可见；
4. 点击播放栏条目后，地图点位和右侧焦点是否同步；
5. 右侧重点情报流是否仍保留，但不再承担播放轴职责；
6. 从首页点详情、返回首页、再切其他菜单是否仍然稳定。
