# 任务：事件中心与告警中心工作台化深化

## 任务背景
在上一轮“界面重构与文档重组”完成后，项目已具备统一的深色工作台主框架，但知识构建中的事件管理页、分析预警中的规则管理页和预警记录页，仍保留较强的传统管理表格形态，与首页、检索、问答、预警概览等重点页面相比，工作台感和主辅层级仍不足。

## 本次任务目标
1. 将事件管理页深化为“事件中心工作台”样式，而不是单一表格页。
2. 将预警规则管理页深化为“规则配置工作台”样式，补齐概览、状态和辅助信息区。
3. 将预警记录页深化为“处置追踪工作台”样式，强化主列表与右侧辅助研判区的关系。
4. 不破坏已有路由、筛选、详情抽屉、钻取跳转和 mock 逻辑。
5. 同步补充关键注释，并在任务结束后回写状态文档。

## 执行范围
### 代码范围
- `src/views/knowledge-build/EventManagement.vue`
- `src/views/analysis-warning/WarningRuleManagement.vue`
- `src/views/analysis-warning/WarningRecordList.vue`
- 如有必要，可补充少量 mock/types 字段与公共样式，但不新增大模块。

### 文档范围
- `doc/rules/当前执行状态.md`
- `doc/rules/文档维护规范.md`
- `doc/tasks/当前任务/任务-事件中心与告警中心工作台化深化.md`
- 需要时同步更新 `doc/README.md` 与 `doc/tasks/待执行任务/后续任务储备.md`

## 设计与实现原则
1. 优先延续现有深色工作台语言，不推翻全局布局。
2. 子页面必须形成“页头概览 / 主工作区 / 辅助信息区 / 详情抽屉”的清晰层级。
3. 尽量复用现有通用页头、面板、图表与全局样式 token。
4. 所有新增的关键计算、筛选、路由钻取逻辑必须补注释。
5. 所有改动必须兼顾 2880、1920、1600、1440 四类宽度。

## 预期产出
1. 事件中心页具备概览指标、事件列表、热点观察、处置建议和详情钻取。
2. 告警规则页具备规则概览、规则列表、运行观察、配置建议和详情/编辑弹层。
3. 预警记录页具备记录概览、记录列表、状态摘要、待处理焦点和详情/闭环操作。
4. 文档体系更新为“当前任务已切换到本任务”。

## 验收方式
1. `npm run build` 通过。
2. 知识构建、分析预警相关路由可正常打开。
3. 事件管理、规则管理、预警记录的筛选、详情、跳转、状态反馈保持可用。
4. 文档读取顺序保持为：规则 -> 当前状态 -> 当前任务。

## 本次改动说明

### 一、页面整改范围
本次重点整改了以下页面：
- `src/views/knowledge-build/EventManagement.vue`
- `src/views/analysis-warning/WarningRuleManagement.vue`
- `src/views/analysis-warning/WarningRecordList.vue`

### 二、核心布局整改动作
1. 事件管理页改造为“事件中心工作台”
   - 增加顶部概览指标区
   - 拆分为主列表区 + 右侧焦点面板 + 分布观察区 + 区域观察区
   - 保留原有详情抽屉、内容跳转、实体跳转和检索/预警联动
2. 预警规则页改造为“规则配置工作台”
   - 增加规则概览指标
   - 增加规则焦点卡、规则类型观察区、最近更新区
   - 补充规则类型和告警等级筛选，不破坏原有新增/编辑/启停/详情能力
3. 预警记录页改造为“处置追踪工作台”
   - 增加记录概览指标
   - 增加当前处置焦点、状态观察、待处理焦点区
   - 保留并强化“标记已处理”的 mock 闭环与详情时间线反馈

### 三、关键稳定性处理
- 统一将三页的 query 参数解析前置为“先重置默认筛选，再消费 drill-down 参数”，避免旧筛选残留
- 统一为焦点对象增加兜底逻辑，当当前筛选结果变化时自动回落到可见对象
- 统一为详情抽屉启用 `destroy-on-close` 和 `append-to-body`，减少页面层级残留风险

### 四、注释补充情况
本次已补充以下关键注释：
- 事件中心筛选状态重置逻辑
- 事件中心 query 参数解析逻辑
- 规则中心默认筛选恢复逻辑
- 规则中心 query 参数钻取逻辑
- 预警记录页默认筛选恢复逻辑
- 预警记录页已处理闭环逻辑
