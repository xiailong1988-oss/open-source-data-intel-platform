# 01-后端工程结构说明

## 一、文档目标
本文档用于说明当前 `backend/` 工程在 Task-12 收尾后的整体结构、模块边界、配置目录和可维护方式，作为后续继续迭代的结构基线。

---

## 二、工程概况
当前后端工程目录位于：

`/backend`

当前工程已经完成：
1. Spring Boot 3 + Java 21 单体模块化骨架
2. Flyway 数据库迁移与初始化数据
3. 首页地区、地图、情报流、概览、热点专题、重点关注接口
4. 情报管理、数据源管理、采集任务管理、数据包管理、交付记录管理接口
5. 轻量登录、当前用户、基础角色控制、操作日志
6. 前后端联调演示数据和文档体系

当前工程仍保持以下原则：
1. 单体模块化，不拆微服务
2. 数据结构统一走 Flyway
3. 接口统一返回 `ApiResponse`
4. 当前阶段优先支撑前端联调和基础后台落地

## 五、模块分层说明
### 5.1 `common`
职责：
1. 放置统一响应、异常、通用配置、认证与日志等跨模块公共能力。
2. 不承载具体业务含义，避免业务模块相互耦合。

当前关键内容：
1. `ApiResponse`、`PageResponse`
2. `BusinessException`、`GlobalExceptionHandler`
3. `LoginRequired`、`RequireRole`、`AuthInterceptor`
4. `OperationLogAspect`
5. `BaseEntity`、`CommonMetaObjectHandler`

### 5.2 `modules/system`
职责：
1. 提供系统基础健康检查和测试接口。
2. 提供登录、退出登录、当前用户接口。
3. 承载 `sys_user`、`sys_role`、`sys_user_role`、`sys_oper_log` 对应实体和服务。

### 5.3 `modules/region`
职责：
1. 提供地区树、地区详情、地区边界查询。
2. 提供地图图层和首页地图点位查询。

说明：
1. 地图点位当前直接复用情报主表中的坐标数据。
2. `biz_region_geo` 当前仍是占位边界数据，不做真实 GIS 计算。

### 5.4 `modules/intel`
职责：
1. 承载情报核心主表、来源、标签和关联关系。
2. 提供后台情报分页、详情、新增、编辑、删除。
3. 提供首页快讯条和主情报流接口。

### 5.5 `modules/dashboard`
职责：
1. 提供首页概览统计、地区态势摘要。
2. 提供热点专题和重点关注对象查询。

### 5.6 `modules/source` 与 `modules/collect`
职责：
1. `source` 负责数据源管理。
2. `collect` 负责采集任务管理。

说明：
1. 当前只实现后台管理与状态展示。
2. 未接入真实调度器、消息流或采集执行引擎。

### 5.7 `modules/packagecenter`
职责：
1. 提供数据包分页、详情、新增、编辑、发布。
2. 提供文件清单查询。
3. 提供交付记录分页、详情和新增。

说明：
1. 当前只预留文件字段和下载地址。
2. 未接入真实文件上传中心和对象存储。

## 六、资源与配置结构说明
### 6.1 `application.yml`
放置公共配置：
1. 应用名
2. 默认环境
3. Flyway 公共配置
4. Jackson 时间格式
5. MyBatis-Plus 逻辑删除配置
6. SpringDoc / Knife4j 配置
7. Actuator 暴露配置
8. 公共日志级别

### 6.2 `application-dev.yml`
用于本地开发与联调：
1. 本地 MySQL 数据源
2. Hikari 连接池
3. 数据库健康检查开启
4. 业务日志级别更详细

### 6.3 `application-prod.yml`
用于部署环境：
1. 通过环境变量读取数据库连接
2. 保持 Flyway 可控执行
3. 日志默认采用生产级别

### 6.4 `application-test.yml`
用于测试环境：
1. 关闭数据源自动配置
2. 关闭 Flyway 自动迁移
3. 保证单元测试不依赖本地数据库

## 八、当前已具备的能力
1. 首页公共接口可直接联调，且已有初始化演示数据。
2. 后台管理接口具备基础登录和 `ADMIN` 角色保护。

5. 工程目录、配置、文档和模块边界已基本稳定，适合继续增量迭代。

---

## 九、当前边界
当前工程仍有明确边界，不在本轮范围内：
1. 不包含复杂 RBAC 权限中心
2. 不包含真实对象存储和文件上传中心
3. 不包含复杂审批流、规则引擎、ES 检索
4. 不包含真实采集调度执行器
5. 不包含真实地区边界 GIS 数据和空间分析能力

---

## 十、维护建议
1. 新增业务能力时优先落到已有模块，不轻易新增横向大模块。
2. 表结构调整必须继续通过 Flyway 增量脚本。
3. 首页接口和后台接口文档建议同步更新 `/doc/backend/` 对应清单。
4. 若后续开始进入可部署阶段，优先补齐 CI、构建缓存和正式环境凭据管理。
