# 06-后端启动说明

## 一、文档目标
本文档说明当前后端在 Task-11 完成后的启动前提、数据库要求、认证方式、演示数据初始化和基础验收步骤。

---

## 二、启动前提
后端目录：`/backend`

环境要求：
1. Java 21
2. Maven 3.9+
3. MySQL 8.x

---

## 三、数据库准备
### 3.1 创建数据库
```sql
CREATE DATABASE IF NOT EXISTS haidian_intel_platform
DEFAULT CHARACTER SET utf8mb4
DEFAULT COLLATE utf8mb4_general_ci;
```

### 3.2 连接参数
默认使用 `dev` 环境，可通过环境变量覆盖：

```bash
SPRING_PROFILES_ACTIVE=dev
DB_URL=jdbc:mysql://127.0.0.1:3306/haidian_intel_platform?useUnicode=true&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false
DB_USERNAME=root
DB_PASSWORD=******
```

---

## 四、启动方式
### 4.1 启动应用
```bash
cd backend
mvn spring-boot:run
```

### 4.2 执行测试
```bash
cd backend
mvn test
```

---

## 六、默认管理员
Flyway 执行 `V9` 后会初始化默认管理员：
1. 用户名：`admin`
2. 初始密码：`Admin@123456`
3. 默认角色：`ADMIN`

## 七、认证方式
### 7.1 登录
调用：

```http
POST /api/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "Admin@123456"
}
```

### 7.2 鉴权头
登录成功后，后续后台管理接口请求头可使用：

```http
Authorization: Bearer <token>
```

或兼容：

```http
X-Auth-Token: <token>
```

### 7.3 登录相关接口
1. `POST /api/auth/login`
2. `POST /api/auth/logout`
3. `GET /api/auth/current-user`

## 九、建议验收步骤
1. 执行 `cd backend && mvn test`
2. 执行 `cd backend && mvn spring-boot:run`
3. 使用默认管理员 `admin / Admin@123456` 调用 `POST /api/auth/login`
4. 在请求头中携带 `Authorization: Bearer <token>`
5. 调用 `GET /api/auth/current-user` 验证登录态
6. 调用 `GET /api/dashboard/overview`、`GET /api/intel/ticker`、`GET /api/map/points` 验证首页演示数据
7. 调用 `GET /api/sources/page`、`GET /api/collect/tasks/page`、`GET /api/packages/page`、`GET /api/delivery-records/page`
8. 调用 `POST /api/auth/logout`
9. 访问 `GET /doc.html` 检查接口文档

## 十、当前说明
1. 当前认证方案采用轻量内存 token 会话，适合单体联调环境，应用重启后需重新登录。
2. 首页联调接口保持公开访问，后台管理接口增加基础角色保护。
3. 当前只做基础角色控制，不提供复杂 RBAC 和菜单权限树。
4. 数据源与采集任务当前以后端管理联调为目标，不包含真实调度引擎。
5. `V10` 初始化数据采用相对时间策略，能更稳定地支撑首页快讯条与热点联调。
6. 复杂任务编排、单点登录、第三方统一认证未在本轮实现。
