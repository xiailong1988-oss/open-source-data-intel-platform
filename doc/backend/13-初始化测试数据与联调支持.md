# 13-初始化测试数据与联调支持

## 一、文档目标
本文档用于说明 Task-11 初始化测试数据后的联调使用方式，重点回答以下问题：
1. 需要启动哪些服务
2. 使用哪个数据库
3. 如何初始化演示数据
4. 前端可联调哪些接口
5. 建议按什么路径做演示验证

---

## 二、需要启动的服务
当前联调只需要以下服务：
1. MySQL 8.x
2. `backend` Spring Boot 服务

说明：
1. 当前不需要额外启动文件中心、对象存储、消息队列或调度中心。
2. 数据包下载地址和文件存储字段仅为占位，不依赖额外服务。

## 三、数据库说明
数据库名称建议使用：

```sql
haidian_intel_platform
```

创建方式：

```sql
CREATE DATABASE IF NOT EXISTS haidian_intel_platform
DEFAULT CHARACTER SET utf8mb4
DEFAULT COLLATE utf8mb4_general_ci;
```

## 四、如何初始化数据
### 4.1 自动初始化方式
当前演示数据通过 Flyway 自动初始化，不需要额外脚本系统。

首次启动或升级到 Task-11 后，应用会自动执行：
1. `V1__create_system_region_map_base_tables.sql`
2. `V2__init_system_region_map_base_data.sql`
3. `V3__create_intel_core_tables.sql`
4. `V4__init_intel_reference_data.sql`
5. `V5__create_dashboard_hotspot_focus_tables.sql`
6. `V6__init_dashboard_hotspot_focus_data.sql`
7. `V7__create_collection_source_task_tables.sql`
8. `V8__create_data_package_delivery_tables.sql`
9. `V9__init_auth_admin_seed.sql`
10. `V10__init_demo_data_for_integration.sql`

### 4.2 启动命令
```bash
cd backend
mvn spring-boot:run
```

### 4.3 重置演示数据建议
如需重新拿到完整预置演示数据，建议使用空库重新启动应用，让 Flyway 从 `V1` 到 `V10` 完整执行。

如果当前数据库已经执行到 `V9`，更新代码后重新启动即可自动执行 `V10`。

## 七、推荐演示路径
### 7.1 首页基础路径
1. 调用 `GET /api/regions/tree`，确认地区树包含 `海淀区`、`青龙桥街道`、`万寿路街道`。
2. 调用 `GET /api/map/layers`，确认返回 `risk/emergency/hotspot/focus` 四类图层。
3. 调用 `GET /api/dashboard/overview`，确认首页概览有统计结果。
4. 调用 `GET /api/dashboard/hotspots` 与 `GET /api/dashboard/focus-targets`，确认卡片列表不为空。

### 7.2 首页情报与地图路径
1. 调用 `GET /api/intel/ticker?limit=6`，确认快讯条返回近期高热度情报。
2. 调用 `GET /api/intel/stream?pageNum=1&pageSize=10`，确认主情报流可分页展示。
3. 调用 `GET /api/map/points`，确认返回带坐标的地图点位。
4. 可进一步使用 `regionId=3004` 或 `regionId=3002` 过滤，验证不同地区联调效果。

### 7.3 后台管理路径
1. 使用 `admin / Admin@123456` 调用 `POST /api/auth/login`。
2. 携带 `Authorization: Bearer <token>` 调用 `GET /api/intel/page?pageNum=1&pageSize=10`。
3. 调用 `GET /api/sources/page?pageNum=1&pageSize=10`，验证运行中、异常、停用状态。
4. 调用 `GET /api/collect/tasks/page?pageNum=1&pageSize=10`，验证运行中、异常、暂停状态。
5. 调用 `GET /api/packages/page?pageNum=1&pageSize=10`，验证草稿和已发布数据包。
6. 调用 `GET /api/packages/11001/files`，验证文件清单展示。
7. 调用 `GET /api/delivery-records/page?pageNum=1&pageSize=10`，验证交付记录列表。

### 7.4 建议使用的示例 ID
1. 地区：`3001`、`3002`、`3004`、`3007`、`3008`
2. 情报：`9001` 到 `9010`
3. 数据源：`10001` 到 `10005`
4. 采集任务：`10101` 到 `10105`
5. 数据包：`11001` 到 `11004`
6. 交付记录：`11201` 到 `11203`

## 八、验收建议
### 8.1 Flyway 验收
```sql
SELECT installed_rank, version, description, success
FROM flyway_schema_history
ORDER BY installed_rank;
```

期望结果：
1. 存在 `V1` 到 `V10`
2. `V10` 状态为成功

### 8.2 关键数据验收
```sql
SELECT COUNT(*) FROM biz_intel_item;

SELECT COUNT(*) FROM biz_focus_target;
SELECT COUNT(*) FROM biz_collection_source;
SELECT COUNT(*) FROM biz_collection_task;
```

### 8.3 页面联调验收重点
1. 首页概览、热点专题、重点关注、快讯条、情报流、地图点位均能直接看到演示数据。
2. 地区切换与地图点位过滤可生效。
3. 登录后台后，数据源、采集任务、数据包、交付记录和情报管理页均有示例数据可展示。
4. 不需要额外人工导数或执行附加初始化脚本。
