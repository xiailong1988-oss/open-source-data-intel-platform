# 04-情报与事件核心模块开发

## 一、文档目的

本文档用于记录 Task-04 的实际落地结果，统一说明情报模块的数据表、接口、核心能力、验收步骤，以及对 Task-05 的支撑范围。

Task-04 只聚焦情报与事件核心模块，不提前扩展：

1. 首页快讯条接口
2. 首页聚合统计
3. 热点专题与重点关注聚合
4. 复杂审核流

## 四、情报模块核心能力说明

### 4.1 基础 CRUD

已支持：

1. 情报新增
2. 情报修改
3. 情报删除
4. 情报详情查询
5. 情报分页查询

### 4.2 分页过滤能力

`GET /api/intel/page` 当前支持以下基础过滤条件：

1. 按地区 `regionId`
2. 按事件类型 `eventTypeId`
3. 按发布时间范围 `startTime ~ endTime`
4. 按关键词 `keyword`
5. 按状态 `status`
6. 按审核状态 `reviewStatus`

关键词当前匹配字段：

1. `title`
2. `summary`
3. `address`

### 4.3 关联回填能力

分页与详情接口均已支持：

1. 来源表 `biz_intel_source` 信息回填
2. 事件类型表 `biz_event_type` 信息回填
3. 标签关联表 `biz_intel_item_tag` 查询
4. 标签主表 `biz_intel_tag` 回填
5. 地区表 `biz_region` 名称回填

### 4.4 写入校验规则

新增与修改时会校验：

1. 地区必须存在且处于启用状态
2. 事件类型必须存在且处于启用状态
3. 来源必须存在且处于启用状态
4. 标签必须存在且处于启用状态
5. 时间范围必须满足 `occurTime <= publishTime`

## 六、验收步骤

### 6.1 启动后端

```bash
cd backend
mvn spring-boot:run
```

### 6.2 核对迁移结果

确认数据库已执行：

1. `V1__create_system_region_map_base_tables.sql`
2. `V2__init_system_region_map_base_data.sql`
3. `V3__create_intel_core_tables.sql`
4. `V4__init_intel_reference_data.sql`

### 6.3 接口验收建议顺序

1. 调用 `GET /api/intel/page`
2. 调用 `POST /api/intel` 新增一条情报
3. 记录返回的 `id`
4. 调用 `GET /api/intel/{id}` 查看详情
5. 调用 `PUT /api/intel/{id}` 修改标题、标签或时间
6. 再次调用 `GET /api/intel/page` 验证筛选结果
7. 调用 `DELETE /api/intel/{id}` 删除

## 七、已知风险与待后续补充项

1. 当前只提供情报主表接口，来源表与标签表的后台管理接口留待后续任务补齐。
2. 时间范围过滤当前统一按 `publishTime` 执行，如后续需要按 `occurTime` 独立过滤，可在不破坏现有接口的前提下扩展参数。
3. 当前未接入复杂审核流，仅保留 `review_status` 字段作为后续扩展位。
4. 当前未补充真实业务测试数据，仍建议在 Task-11 统一准备完整联调样本。

## 八、为 Task-05 提供的基础能力

Task-05 可以直接复用本任务输出的以下底座：

1. 情报分页查询能力，可直接作为快讯条和情报流的数据来源基础
2. 情报详情查询能力，可支撑详情抽屉或详情页
3. 地区、事件类型、来源、标签联查能力，可直接回填前端展示字段
4. 发布时间、发生时间、热度、重点关注等字段，可直接支撑排序与筛选
5. 经纬度与地址字段，可为地图点位接口提供原始数据来源
