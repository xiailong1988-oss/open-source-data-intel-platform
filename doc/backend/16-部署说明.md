# 16-部署说明

## 一、文档目标
本文档用于说明当前后端的最小可运行部署方式，方便测试环境、演示环境或后续正式环境继续落地。

---

## 二、部署前提
部署当前后端需要以下基础环境：
1. Java 21
2. Maven 3.9+
3. MySQL 8.x

当前不依赖：
1. Redis
2. MQ
3. Elasticsearch
4. 对象存储
5. 第三方认证中心

---

## 三、数据库准备
### 3.1 创建数据库
```sql
CREATE DATABASE IF NOT EXISTS haidian_intel_platform
DEFAULT CHARACTER SET utf8mb4
DEFAULT COLLATE utf8mb4_general_ci;
```

### 3.2 说明
1. 应用启动后会自动通过 Flyway 执行 `V1` 到 `V10`
2. 默认管理员和联调演示数据也会自动初始化

---

## 四、构建方式
### 4.1 Maven 构建
```bash
cd backend
mvn clean package -DskipTests
```

### 4.2 产物位置
默认产物位于：

```text
backend/target/backend-0.0.1-SNAPSHOT.jar
```

---

## 五、启动方式
### 5.1 环境变量方式
```bash
export SPRING_PROFILES_ACTIVE=prod
export DB_URL='jdbc:mysql://127.0.0.1:3306/haidian_intel_platform?useUnicode=true&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false'
export DB_USERNAME='root'
export DB_PASSWORD='your-password'
export SERVER_PORT='8080'
```

### 5.2 JAR 启动
```bash
java -jar target/backend-0.0.1-SNAPSHOT.jar
```

或：

```bash
java -jar target/backend-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

---

## 六、部署后校验
部署成功后建议依次校验：
1. `GET /api/health`
2. `GET /actuator/health`
3. `GET /doc.html`
4. `POST /api/auth/login`
5. `GET /api/dashboard/overview`
6. `GET /api/sources/page`

说明：
1. 第 4 步之后的后台接口需要携带登录 token
2. 若只是验证首页展示，可直接访问公共接口

---

## 七、默认账号
当前默认管理员账号由 Flyway 初始化：
1. 用户名：`admin`
2. 密码：`Admin@123456`
3. 角色：`ADMIN`

---

## 八、当前部署边界
当前部署方案是单体后端的最小可运行方案，明确不包含：
1. 多节点 session 共享
2. 容器编排模板
3. Nginx 反向代理示例
4. 文件中心和对象存储挂载
5. 自动化 CI/CD 流水线

---
