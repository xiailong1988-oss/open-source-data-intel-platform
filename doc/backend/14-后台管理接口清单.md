# 14-后台管理接口清单

## 一、文档目标
本文档汇总当前后台管理相关接口、鉴权要求和适用页面，方便后台前端联调与后续维护。

---

## 二、统一约定
### 2.1 返回结构
所有后台接口统一返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "timestamp": "2026-04-09T10:00:00"
}
```

### 2.2 登录与鉴权
除登录接口外，后台管理接口当前统一要求：
1. 先调用 `POST /api/auth/login`
2. 在请求头中携带 `Authorization: Bearer <token>`
3. 当前后台接口默认要求角色 `ADMIN`

### 2.3 当前轻量权限策略
1. 仅实现基础登录态校验和角色校验
2. 不实现复杂 RBAC 权限树
3. 不实现按钮级权限或菜单权限中心

### 2.4 当前已补充的关键录入校验
当前后台录入接口已补充部分基础格式校验，避免明显不合理数据直接入库：
1. `contactPhone`：增加手机号/服务电话基础格式校验
2. `sourceUrl`：要求带协议头或方案头，例如 `https://`、`jdbc:`、`mqtt:`、`smb://`
3. `downloadUrl`：若传值，则要求带协议头
4. `statMonth`：要求符合 `yyyy-MM`

## 三、认证接口
### 3.1 登录
`POST /api/auth/login`

用途：
1. 使用用户名和密码换取 token
2. 返回当前用户信息和角色信息

### 3.2 当前用户
`GET /api/auth/current-user`

用途：
1. 获取当前登录用户基础信息
2. 用于前端回显用户名、角色和登录态

### 3.3 退出登录
`POST /api/auth/logout`

用途：
1. 注销当前 token 会话

## 九、当前后台接口保护方式
### 9.1 注解方式
当前主要通过以下注解保护后台接口：
1. `@LoginRequired`
2. `@RequireRole("ADMIN")`

### 9.2 保护范围
当前已加保护的后台接口包括：
1. 情报管理接口
2. 数据源管理接口
3. 采集任务管理接口
4. 数据包管理接口
5. 交付记录接口
6. 当前用户与退出登录接口

### 9.3 当前不受后台鉴权影响的接口
以下接口仍保持公开访问：
1. 首页公共接口
2. 地区与地图公共接口
3. 健康检查接口
4. 接口文档访问路径

## 十、联调建议
1. 先调用 `POST /api/auth/login` 获取 token。
2. 再调用 `GET /api/auth/current-user` 验证登录态。
3. 后台管理列表页联调建议优先验证 `GET /api/intel/page`。
4. 后台管理列表页联调建议优先验证 `GET /api/sources/page`。
5. 后台管理列表页联调建议优先验证 `GET /api/collect/tasks/page`。
6. 后台管理列表页联调建议优先验证 `GET /api/packages/page`。
7. 后台管理列表页联调建议优先验证 `GET /api/delivery-records/page`。
8. 若出现 `unauthorized`，优先检查 token 是否缺失或已过期。
9. 若出现 `forbidden`，优先检查当前用户是否具备 `ADMIN` 角色。
