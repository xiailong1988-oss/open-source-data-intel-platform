# 08-地图点位与首页概览接口开发

## 一、任务目标
基于已有情报主表，为首页地图点位、顶部概览卡片和地区态势摘要提供可直接联调的后端接口。

本轮严格限定在 Task-06 范围内，不扩展热点专题、重点关注完整模块、复杂统计任务调度或缓存优化。

---

## 二、本次完成内容
### 2.1 地图点位接口
新增接口：
`GET /api/map/points`

支持条件：
1. `regionId`
2. `eventTypeId`
3. `startTime`
4. `endTime`

返回字段：
1. `id`
2. `title`
3. `regionId`
4. `region`
5. `eventTypeId`
6. `type`
7. `category`
8. `lng`
9. `lat`
10. `coords`
11. `heat`
12. `importanceLevel`
13. `riskLevel`
14. `focusFlag`
15. `time`
16. `summary`

实现说明：
1. 仅返回首页可见且已审核通过的数据
2. 仅返回带坐标的数据
3. `coords` 按 `[lng, lat]` 输出，便于前端地图直接消费
4. `time` 优先使用事件发生时间，缺失时回退到发布时间

---

### 2.2 首页概览统计接口
新增接口：
`GET /api/dashboard/overview`

支持条件：
1. `regionId`
2. `eventTypeId`
3. `startTime`
4. `endTime`

返回字段：
1. `regionId`
2. `regionName`
3. `totalEventCount`
4. `riskEventCount`
5. `emergencyEventCount`
6. `hotspotEventCount`
7. `focusCount`
8. `latestUpdateTime`

统计口径：
1. 总事件数：当前过滤条件下的首页可见情报总数
2. 风险事件数：事件分类为 `risk`
3. 突发事件数：事件分类为 `emergency`
4. 热点事件数：事件分类为 `hotspot`
5. 重点关注数：`focusFlag=1` 或事件分类为 `focus` 的近似统计
6. 最近更新时间：当前过滤结果中的最新情报更新时间

---

### 2.3 地区态势摘要接口
新增接口：
`GET /api/dashboard/region-summary`

支持条件：
1. `regionId`
2. `eventTypeId`
3. `startTime`
4. `endTime`

返回字段：
1. `regionId`
2. `regionName`
3. `summary`
4. `totalEventCount`
5. `riskEventCount`
6. `emergencyEventCount`
7. `hotspotEventCount`
8. `focusCount`
9. `latestUpdateTime`

实现说明：
1. 按当前筛选条件生成一句话摘要
2. 无数据时返回可直接展示的空态摘要
3. 复用首页概览统计口径，避免不同卡片统计口径不一致

## 三、小范围结构修正说明
本轮未调整数据库结构，也未引入新表。

仅做了一个小范围结构补充：
1. 在 `region` 模块新增地图点位接口与服务，保持 `/api/map/*` 命名空间清晰
2. 在 `dashboard` 模块新增首页概览服务，避免把统计逻辑继续堆进 `intel` 模块

这样做的原因是：
1. 现有结构已能支撑 Task-06
2. 但如果继续把地图点位和概览统计都写入 `intel`，首页职责会变得混乱
3. 当前拆分仍然是单体模块化范围内的小修正，不改变整体方向

## 五、给前端的接入说明
### 5.1 地图点位
前端请求：
`GET /api/map/points`

接入建议：
1. 地区切换时传 `regionId`
2. 事件类型切换时传 `eventTypeId`
3. 时间范围切换时传 `startTime/endTime`
4. 地图渲染直接使用 `coords`
5. 点位详情跳转直接使用 `id`

### 5.2 顶部概览卡片
前端请求：
`GET /api/dashboard/overview`

接入建议：
1. 顶部卡片直接绑定各类统计字段
2. 地区切换时与地图点位保持相同过滤条件
3. `latestUpdateTime` 可直接显示为数据刷新提示

### 5.3 地区态势摘要
前端请求：
`GET /api/dashboard/region-summary`

接入建议：
1. 将 `summary` 直接绑定到一句话摘要区域
2. 卡片侧边的小数字可直接复用基础统计字段
