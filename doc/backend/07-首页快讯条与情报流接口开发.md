# 07-首页快讯条与情报流接口开发

## 一、文档目的

本文档用于记录 Task-05 的实际落地结果，说明首页快讯条与主情报流接口的能力边界、返回字段、前端接入方式和验收步骤。

1. 首页快讯条接口
2. 首页主情报流接口

## 二、新增接口说明

### 2.1 快讯条接口

请求地址：

`GET /api/intel/ticker`

说明：

- 返回近期、高优先级、适合横向播报的数据
- 仅返回 `status=1` 且 `reviewStatus=1` 的情报
- 默认限制条数为 `10`

支持参数：

| 参数名 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `regionId` | `long` | 空 | 按地区过滤 |
| `eventTypeId` | `long` | 空 | 按事件类型过滤 |
| `keyword` | `string` | 空 | 按标题/摘要/地址过滤 |
| `limit` | `int` | `10` | 返回数量，最大 `30` |

排序逻辑：

1. `focusFlag desc`
2. `importanceLevel desc`
3. `heatScore desc`
4. `publishTime desc`
5. `id desc`

### 2.2 主情报流接口

请求地址：

`GET /api/intel/stream`

说明：

- 返回首页主情报流分页数据
- 仅返回 `status=1` 且 `reviewStatus=1` 的情报
- 按时间倒序分页

支持参数：

| 参数名 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `pageNum` | `long` | `1` | 页码 |
| `pageSize` | `long` | `10` | 每页条数 |
| `regionId` | `long` | 空 | 按地区过滤 |
| `eventTypeId` | `long` | 空 | 按事件类型过滤 |
| `keyword` | `string` | 空 | 按标题/摘要/地址过滤 |

排序逻辑：

1. `publishTime desc`
2. `id desc`

## 三、返回字段说明

快讯条和主情报流当前统一输出以下核心字段：

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| `detailId` | `long` | 情报详情 ID，用于打开详情 |
| `title` | `string` | 情报标题 |
| `time` | `datetime` | 当前取情报发布时间 `publishTime` |
| `region` | `string` | 地区名称 |
| `type` | `string` | 事件类型名称 |
| `source` | `string` | 来源名称 |
| `summary` | `string` | 摘要 |
| `tags` | `string[]` | 标签名称列表 |

快讯条返回示例：

```json
[
  {
    "detailId": 9001,
    "title": "校园周边群体活动",
    "time": "2026-04-07T10:00:00",
    "region": "海淀区",
    "type": "群体活动",
    "source": "人工录入",
    "summary": "接到街道反馈，校园周边出现聚集活动。",
    "tags": ["校园安全", "群体活动"]
  }
]
```

主情报流返回示例：

```json
{
  "records": [
    {
      "detailId": 9001,
      "title": "校园周边群体活动",
      "time": "2026-04-07T10:00:00",
      "region": "海淀区",
      "type": "群体活动",
      "source": "人工录入",
      "summary": "接到街道反馈，校园周边出现聚集活动。",
      "tags": ["校园安全", "群体活动"]
    }
  ],
  "total": 1,
  "pageNum": 1,
  "pageSize": 10
}
```

## 四、前端如何接入快讯条

推荐调用方式：

```http
GET /api/intel/ticker?regionId=3001&limit=8
```

前端接入建议：

1. 页面初始化时直接请求一次快讯条接口
2. 将返回数组直接绑定到横向播报组件
3. 使用 `title`、`time`、`type`、`region`、`source` 渲染播报内容
4. 点击单条快讯时使用 `detailId` 跳转或打开详情抽屉

如果需要按地区或事件类型联动：

1. 地区切换时带上 `regionId`
2. 事件类型切换时带上 `eventTypeId`
3. 搜索框输入时带上 `keyword`

---

## 五、前端如何接入主情报流

推荐调用方式：

```http
GET /api/intel/stream?pageNum=1&pageSize=10&regionId=3001
```

前端接入建议：

1. 页面初始化时请求第一页
2. 列表组件直接绑定 `data.records`
3. 分页器绑定 `total`、`pageNum`、`pageSize`
4. 列表卡片展示 `title`、`summary`、`time`、`region`、`type`、`source`、`tags`
5. 点击卡片时使用 `detailId` 查询详情

筛选联动建议：

1. 地区筛选绑定 `regionId`
2. 类型筛选绑定 `eventTypeId`
3. 搜索框绑定 `keyword`
4. 翻页时仅调整 `pageNum` 和 `pageSize`

## 六、验收方式

### 6.1 启动服务

```bash
cd backend
mvn spring-boot:run
```

### 6.2 快讯条接口验收

1. 请求 `GET /api/intel/ticker`
2. 验证返回为数组
3. 验证字段包含 `detailId`、`title`、`time`、`region`、`type`、`source`、`summary`、`tags`
4. 带 `regionId`、`eventTypeId`、`keyword` 再次请求，验证过滤有效

### 6.3 主情报流接口验收

1. 请求 `GET /api/intel/stream?pageNum=1&pageSize=10`
2. 验证返回结构包含 `records`、`total`、`pageNum`、`pageSize`
3. 验证 `records` 中字段完整
4. 验证列表按 `time` 倒序展示
5. 带 `regionId`、`eventTypeId`、`keyword` 再次请求，验证过滤有效

### 6.4 空数据验收

1. 使用不存在结果的 `keyword` 请求 `ticker`
2. 验证返回空数组 `[]`
3. 使用不存在结果的 `keyword` 请求 `stream`
4. 验证返回空分页结构，而不是报错

## 七、已知风险与待后续补充项

1. 快讯条“近期”当前固定按最近 `72` 小时判断，如后续需要后台可配置化，可在后续任务扩展。
2. `time` 当前统一取 `publishTime`，如后续首页设计改为展示 `occurTime`，只需小范围调整映射逻辑。
3. 当前主情报流未接入缓存与复杂排序策略，优先保证前端联调可用和结果稳定。
4. 当前仍依赖已有情报基础数据，若测试库样本不足，需要在后续任务统一补充初始化联调数据。
