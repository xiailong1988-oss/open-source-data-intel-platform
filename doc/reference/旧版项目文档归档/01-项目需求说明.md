# 项目需求说明

## 建设背景

平台面向“多源开源数据与业务数据的统一分析”场景，目标是把数据接入、治理、知识构建、检索问答、预警和报告整合到一套统一界面中。当前阶段主要服务于方案演示、需求确认、内部评审和后续真实开发准备。

## 用户使用目标

典型用户进入平台后，希望完成以下动作：

- 查看当前整体态势，快速识别重点区域、热点专题和风险对象。
- 管理数据源和采集任务，确认数据是否持续进入平台。
- 查看治理质量、标准化结果、实体和事件结构，评估数据到知识的转换效果。
- 用智能检索和智能问答快速定位信息、汇总结论并继续追踪对象。
- 在分析预警模块中发现异常、查看规则、跟踪记录并完成闭环处置。
- 在报告中心中查看和演示 AI 辅助分析报告。

## 当前阶段需求范围

当前项目已经覆盖以下原型范围：

- 综合态势首页
- 地图总览模块
- 数据接入
- 数据治理
- 知识构建
- 智能检索
- 智能问答
- 分析预警
- 报告中心
- 系统管理

## 核心业务主线

当前系统的业务主线如下：

1. 数据接入：接入外部数据源、配置采集方式、查看采集任务和执行日志。
2. 数据治理：清洗、去重、标准化、质量评估，并形成标准化数据列表。
3. 知识构建：从治理后的内容中抽取实体、事件、关系和标签，形成知识资产。
4. 智能检索 / 智能问答：让用户围绕关键词、语义、对象关系进行检索和问答探索。
5. 分析预警：根据规则和趋势识别风险，形成告警记录与处置闭环。
6. 报告输出：把分析结果沉淀为正式报告，用于演示或研判汇报。
7. 系统管理：管理用户、角色权限和系统配置。

## 首页综合态势的需求意义

首页不是单纯的 BI 看板，而是整个平台的综合入口。首页需要同时承载：

- 统计卡片：让用户快速看到平台当前规模与活跃度。
- 地图总览：提供空间态势感知能力，让平台具备“区域、对象、风险、专题”的空间视角。
- 辅助图表：补充来源结构、增长趋势、主题热度和数据类型分布。
- 预警、任务、专题、报告列表：让用户能从总览直接进入具体业务对象。
- 快捷入口：缩短从首页到重点模块的操作路径。

## 模块需求说明

### 智能检索

解决“如何快速从多维数据中找到相关内容”的问题。需要支持关键词、语义模式、标签、来源、地域、风险等级等条件组合，并从结果继续钻取到内容、实体和事件。

### 智能问答

解决“如何把知识和检索结果转成业务问答”的问题。页面要体现分析型问答，不是娱乐聊天。答案需要带依据来源、相关实体和推荐追问。

### 分析预警

解决“如何发现风险、配置规则、跟踪处置”的问题。需要有概览、规则管理和记录管理三层结构，并形成从发现到处理的闭环感。

### 报告中心

解决“如何把分析结果正式输出”的问题。报告不仅要能看，还要能回溯事件、实体和引用内容。

### 系统管理

解决“平台能力如何被管理和配置”的问题，虽然当前不接真实权限系统，但页面需要体现用户、角色和配置管理的完整性。

## 当前哪些功能是 mock 演示

- 所有列表、详情、图表、地图点位和问答答案均来自 mock 数据。
- 智能问答没有接真实模型，采用预置答案和 fallback mock 生成。
- 智能检索没有接真实搜索引擎，通过 mock 数据和前端筛选模拟。
- 地图没有接真实 GIS 服务，采用 ECharts + SVG 注册图模拟空间区域。
- 报告导出、下载、生成、全屏查看等都是前端假动作反馈。

## 当前为未来真实落地预留的内容

- 路由结构、模块边界、页面层级已经稳定，适合后续接接口。
- mock 数据按模块拆分，后续可以逐步替换成 API 层。
- query 参数驱动的页面跳转与筛选逻辑，未来可以无缝衔接真实查询参数。
- 地图点位、区域、图层和右侧面板已经有基础数据结构，后续可接真实 GIS 数据。

## 当前不做的内容

- 不接真实后端接口
- 不接真实数据库
- 不接真实模型
- 不接真实搜索服务
- 不接真实 GIS 服务
- 不实现真实登录与鉴权
- 不实现复杂规则引擎

这些内容只有在进入明确的新阶段后才考虑接入。
