# 通用组件说明

本文件用于沉淀当前项目已经抽离出的通用组件，重点服务于 `12-Codex执行任务拆分文档.md` 中 `Phase 1：基础重构` 的组件沉淀目标。

## 1. 文档用途

- 说明当前哪些组件已经作为通用能力沉淀
- 说明组件适用场景、输入输出和使用边界
- 避免同类组件在不同页面重复造轮子
- 为后续页面改造提供统一复用依据

## 2. 当前已沉淀的重点组件

## 2.1 [PageSectionHeader.vue](E:/codex-test/open-source-data-intel-platform/src/components/common/PageSectionHeader.vue)

### 用途

统一页面分区标题区，承接标题、副标题、眉标和右侧操作区。

### 适用场景

- 首页分区卡片头部
- 检索、问答、预警、报告等页面中的板块标题
- 后续专题工作台、事件中心、告警中心中的区块标题

### 当前已接入

- 综合态势首页
- 智能检索页
- 首页快捷入口组件

### 使用约束

- 如果只是普通卡片标题，优先复用该组件，不要每个页面单独写一套头部结构。
- 如果需要按钮操作，应通过 `actions` 插槽扩展，而不是另起一套布局。

## 2.2 [ViewStatePanel.vue](E:/codex-test/open-source-data-intel-platform/src/components/common/ViewStatePanel.vue)

### 用途

统一承载加载态、空态、异常态和重试动作。

### 适用场景

- 检索结果为空
- 页面数据加载中
- 异常状态需要重试
- 卡片或列表局部无数据解释

### 当前已接入

- 智能检索页
- 首页快捷入口组件

### 使用约束

- 新增页面如果存在 `loading / empty / error` 三类状态，优先接入该组件。
- 如果页面需要自定义操作按钮，通过 `actions` 插槽补充，不要重新实现一套空态区。

## 2.3 [GlobalCommandBar.vue](E:/codex-test/open-source-data-intel-platform/src/layout/components/GlobalCommandBar.vue)

### 用途

作为全站顶部统一命令入口，承接智能检索 / 智能问答的统一输入。

### 适用场景

- 顶部统一入口
- 后续专题工作台顶部搜索区
- 需要复用“模式切换 + 输入 + 推荐问题”的场景

### 当前已接入

- 主布局顶部栏

### 使用约束

- 当前组件只负责输入与模式选择，不直接决定跳转目标。
- 页面跳转、query 注入和上下文拼装应由上层布局或容器组件负责。

## 2.4 [MetricStatCard.vue](E:/codex-test/open-source-data-intel-platform/src/components/dashboard/MetricStatCard.vue)

### 用途

统一首页与概览页中的指标卡样式、数值展示和点击态。

### 适用场景

- 综合态势指标卡
- 数据治理、知识构建、分析预警概览中的统计卡片

### 使用约束

- 指标卡需要支持点击钻取时，应统一通过外层传入点击事件，而不是把业务跳转写死在组件内部。

## 2.5 [ChartCard.vue](E:/codex-test/open-source-data-intel-platform/src/components/charts/ChartCard.vue)

### 用途

统一 ECharts 图表外层卡片、标题区和点击事件透传。

### 适用场景

- 首页图表
- 各概览页统计图表
- 后续专题工作台趋势、分布、排行类图表

### 使用约束

- 图表 click drill down 应由页面层处理，组件层负责透传，不要把业务路由耦合进图表组件内部。

## 2.6 [SearchResultCard.vue](E:/codex-test/open-source-data-intel-platform/src/components/search/SearchResultCard.vue)

### 用途

统一智能检索结果的标题、摘要、标签、来源、风险等级和详情操作展示。

### 适用场景

- 智能检索页结果列表
- 后续专题检索结果页

### 使用约束

- 若未来扩展检索列表页，应优先在该组件上做能力增强，而不是复制一份新结果卡片。

## 3. 当前基础组件能力结论

当前项目已经形成一组可以继续扩展的基础组件能力：

- 页面区块标题统一
- 状态展示统一
- 顶部统一入口已形成
- 指标卡和图表卡已有基础抽象
- 检索结果卡具备复用基础

这套能力已经能够支撑 `Phase 1` 的基础重构继续推进，也能为 `Phase 2：首页工作台` 和后续专题工作台改造提供复用基础。

## 4. 后续组件沉淀建议

后续建议继续沉淀以下组件：

- 页面级容器组件
- 统一筛选栏组件
- 风险等级标签组件
- 时间范围切换组件
- 地图区块容器组件
- 事件 / 告警 / 报告的统一详情块组件

## 5. 使用原则

- 能复用就不要重复实现。
- 组件的职责应聚焦在展示结构和交互承载，不应直接耦合业务路由。
- 新增基础组件后，必须补注释并更新本文件。
