# 项目总览

## 项目名称

开源数据智能分析平台

## 项目定位

这是一个以“开源数据汇聚、治理、知识构建、智能检索问答、风险预警与报告输出”为主线的 Web 管理平台原型。当前项目重点不是交付真实生产系统，而是先搭建一套结构完整、页面完整、联动完整、可持续扩展的演示平台。

## 项目要解决的问题

- 当前多源数据接入、治理、知识构建和分析结果展示通常分散在不同系统中，缺乏统一入口。
- 传统 BI 看板只能看指标，缺少从总览到对象、从对象到关联信息的业务闭环。
- 智能检索、智能问答、预警和报告若彼此割裂，会削弱“智能分析平台”的整体价值感。
- 在真实后端、真实模型和真实 GIS 尚未落地前，需要一个高保真前端原型承接方案演示、需求沟通和后续开发。

## 项目整体目标

- 搭建一套统一后台布局的前端平台原型。
- 形成“数据接入 -> 数据治理 -> 知识构建 -> 智能检索/问答 -> 分析预警 -> 报告输出 -> 系统管理”的完整产品主线。
- 让首页不仅能做静态展示，还能作为整个平台的总览入口和联动入口。
- 用 mock 数据模拟真实业务对象、列表、详情、关联关系和地图点位，为后续真实接入预留空间。

## 平台类型

当前项目属于“企业/政府级智能分析平台原型”，同时具备以下特征：

- 数据管理平台：提供数据源管理、采集任务、治理列表和治理概览。
- 知识中台平台：提供实体管理、事件管理和知识构建概览。
- 智能分析平台：提供智能检索、智能问答、分析预警和报告中心。
- 空间态势感知平台：首页已加入地图总览，支持专题图层、点位联动和区域联动。

## 核心模块角色

- 首页综合态势：整个平台的总览入口，承载统计卡片、地图、图表、业务列表和快捷入口。
- 地图总览模块：首页的空间态势核心区，负责把预警、科研、产业等主题映射到空间分布视角。
- 智能问答：展示“基于知识和检索结果生成业务答案”的平台智能能力。
- 分析预警：负责风险发现、规则配置、预警记录和处置闭环。
- 报告中心：承担分析结果输出和正式演示载体。
- 数据接入 / 治理 / 知识构建：构成平台能力底座，为检索、问答、预警和报告提供业务来源。

## 当前项目阶段

当前项目处于“前端原型 + 高保真演示 + 可扩展开发基线”阶段。

这意味着：

- 页面结构、模块布局和交互闭环已经具备演示价值。
- 核心模块之间已经建立基础跳转和联动关系。
- 数据、搜索、问答、预警、报告、地图等能力仍然以 mock 演示为主。
- 代码结构与文档体系需要持续沉淀，为后续接真实服务做好准备。

## 当前建设策略

当前采用“mock 数据 + 前端演示优先”的建设策略，原因如下：

- 可以快速验证产品结构和展示逻辑。
- 可以提前沉淀菜单、路由、页面层级、组件复用和联动关系。
- 可以把后续真实接入的边界和扩展点留在架构层，而不是一开始就绑死到某个后端实现。
