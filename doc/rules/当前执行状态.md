## 本轮任务结论
用户已将首页改造要求升级为“区域态势驾驶舱”级别，当前首页虽然已有地图、情报流、底部摘要矩阵和基础拖拽机制，但仍存在结构性偏差：
1. 地图尚未形成真正的首屏第一核心区域；
2. 缺少“海淀区全域 + 重点地区”切换能力；
3. 快讯条与主情报流尚未形成双层信息结构；
4. 首页所有模块还没有统一响应 `currentRegion`、`currentMapLayer`、`layoutState`；
5. 非地图面板的拖拽与缩放能力仍不完整，且缺少自适应重排；
6. 当前四区骨架更偏静态展示，还不够像真正可用的区域态势驾驶舱。

本轮执行重点切换为：
1. 首页骨架重排为“地图主位 + 地区切换 + 快讯条 + 联动面板 + 六大业务板块”；
2. 建立统一首页状态管理与地区配置；
3. 实现地区切换驱动地图、统计、快讯、情报流和事件模块联动；
4. 改造地图为程序化视角控制，禁用用户自由拖拽与缩放；
5. 升级主情报流可读性与滚动体验；
6. 完善非地图模块拖拽、缩放、布局持久化和自适应重排。
