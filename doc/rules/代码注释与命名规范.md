# 代码注释与命名规范

本文件用于统一当前项目的命名方式和注释规则。

## 1. 注释规则

1. 所有新增函数必须写注释。
2. 所有新增核心组件必须写组件说明注释。
3. 关键布局文件、关键筛选逻辑、关键联动逻辑、关键状态逻辑必须写注释。
4. 注释优先说明“做什么”和“为什么这样做”，避免废话式注释。
5. 复杂 `watch`、`computed`、`route query` 解析、地图联动、图表 drill down 必须写注释。

## 2. 命名规则

1. 组件名使用明确语义，不允许使用含糊缩写。
2. 样式类名优先使用“模块名 + 结构名”的形式，避免全局污染。
3. 变量名必须体现业务含义，不允许出现无语义的 `data1`、`temp`、`box` 等命名。
4. 布局类和通用样式类应突出用途，例如：`platform-page-shell`、`platform-panel-card`。

## 3. 改造期补充规则

1. 所有本轮新增或重构的关键布局文件必须补注释。
2. 所有新建的通用组件都要写明适用场景。
3. 界面重构期间如果替换了旧布局结构，必须在组件注释里说明替换原因。

## 4. 示例要求

- 函数注释：说明输入、输出和存在原因
- 组件注释：说明该组件承载哪类页面结构
- 布局注释：说明为何采用某种 grid / flex 结构，以及解决了什么问题
